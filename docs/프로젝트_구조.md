# 6.3 전국지방선거 인터랙티브 선거 정보 지도 - 프로젝트 구조

> 최종 업데이트: 2026-03-18 | 배포: https://korea-local-eletion.pages.dev

---

## 1. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 프론트엔드 | 바닐라 JS + D3.js v7 + Chart.js v4 | 지도·차트·UI |
| 스타일 | CSS3 (반응형) + Font Awesome + Noto Sans KR | 디자인 |
| 데이터 | JSON (정적 호스팅) | Git 추적 가능 |
| 자동화 | Python 3.11 + GitHub Actions | 일일 배치 |
| LLM | Claude Haiku (개요), Gemini (팩트체크/PDF) | 자동 분석 |
| API | 네이버 뉴스, NESDC, NEC(선관위) | 외부 데이터 |
| 배포 | Cloudflare Pages | 정적 호스팅 |

## 5. 데이터 흐름

### 5.1 초기화 (App.init)

```
DOMContentLoaded
  → DataLoader: data/static/*.json 11개 병렬 로드 → ElectionData에 merge
  → ElectionData: 후보자/여론조사/역대결과 등 15개 JSON 순차 로드
  → LocalMediaPool + LocalMediaRegistry 로드 → 병합
  → MapModule.init(): TopoJSON 로드 → D3 렌더링
  → 로딩 화면 제거
```

### 5.2 지역 선택

```
사용자: 지도 클릭 or 검색
  → MapModule: 해당 지역 하이라이트 + 줌
  → App.selectRegion(regionKey, districtName, electionType)
  → 우측 패널 열기
  → 현재 탭 렌더링 (overview/polls/candidates/news/history)
  → 각 탭이 ElectionData에서 해당 지역 데이터 조회
```

### 5.3 자동화 파이프라인 (백엔드 → 프론트엔드)

```
GitHub Actions (매일)
  │
  ├─ 07:00 KST ─ 선거 개요 업데이트
  │   네이버 뉴스 검색 → Claude Haiku로 narrative 생성
  │   → election_overview.json 업데이트 → git push
  │
  ├─ 09:00 KST ─ 후보자 팩트체크
  │   NEC API + 네이버 뉴스 → Gemini로 분석
  │   → governor/superintendent/mayor_candidates.json
  │   → 교차검증 (cross_validate.py)
  │   → git push
  │
  └─ 수시 ─ 여론조사 수집
      NESDC 스크래핑 → PDF 다운로드 → Gemini 파싱
      → polls/polls.json 업데이트 → git push
      │
      ↓
프론트엔드: JSON fetch → UI 렌더링
```

## 6. 전역 객체 API 요약

### ElectionData (data.js)

```javascript
// 정당
ElectionData.getPartyName('democratic')   // '더불어민주당'
ElectionData.getPartyColor('ppp')          // '#E61E2B'

// 지역
ElectionData.regions['seoul']              // { name, population, voters, ... }
ElectionData.getSubRegions('gyeonggi')     // 기초자치단체 목록

// 후보자
ElectionData.getCandidates(regionKey, electionType)
ElectionData.getGovernorCandidates(regionKey)
ElectionData.getMayorCandidates(regionKey)

// 여론조사
ElectionData.getLatestPollsForDisplay(regionKey, electionType, districtName)
ElectionData.getPollTrends(regionKey, electionType, districtName)

// 역대 선거
ElectionData.getHistoricalData(regionKey)
ElectionData.getMayorHistoricalData(regionKey, districtName)

// 선거 개요
ElectionData.getElectionOverview(regionKey, electionType, districtName)

// 비동기 로드
await ElectionData.loadPollsData()
await ElectionData.loadMayorCandidates()
await ElectionData.loadByElectionData()
// ... 총 15개
```

### MapModule (map.js)

```javascript
MapModule.init()                                    // 초기화
MapModule.selectRegion(regionKey)                   // 광역 선택
MapModule.selectDistrict(regionKey, districtName)   // 기초 선택
MapModule.setElectionType(electionType)             // 선거유형 색상
MapModule.switchToProvinceMap()                     // 전국 보기
MapModule.getCurrentMode()                          // 'province'|'district'|'subdistrict'
```

### ChartsModule (charts.js)

```javascript
ChartsModule.renderPollBarChart(poll, canvasId)
ChartsModule.renderPollTrendChart(regionKey, electionType, canvasId)
ChartsModule.destroyCharts()
```

### NewsFilterConfig (news_filters.js)

```javascript
NewsFilterConfig.majorNewsHosts          // 주요 언론사 도메인 목록
NewsFilterConfig.filterAndRankNews(articles, regionKey)
  // → 신뢰도 + 최신성 + 관련성 종합 점수로 정렬
```

### DataLoader (data-loader.js)

```javascript
await DataLoader.applyToElectionData(ElectionData)  // 정적 JSON → merge
await DataLoader.loadJSON(filename)                  // 개별 파일
```

### update-candidates.yml 실행 순서

```
1. detect_byelection_news.py      → 재보궐 뉴스 감지
2. detect_byelections.py --apply  → NEC API 재보궐 선거구 감지
3. sync_byelection_winners.py     → 재보궐 당선자 동기화
4. factcheck_candidates.py        → 광역단체장 팩트체크
5. factcheck_superintendent.py    → 교육감 팩트체크
6. factcheck_mayor.py             → 기초단체장 팩트체크
7. factcheck_byelection.py        → 재보궐 후보 팩트체크
8. collect_mayor_pledges.py       → 기초단체장 공약 수집
9. sync_overview_candidates.py    → 개요에서 후보 동기화
10. cross_validate.py --fix       → 교차검증 + 자동수정
```
