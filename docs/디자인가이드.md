# 6.3 선거정보지도 디자인 가이드

> 방향: 토스/카카오 스타일 — 깔끔, 미니멀, 정보 위계 확실
> 이 파일을 프로젝트 루트 또는 docs/에 넣으면
> Claude Code가 frontend-design 플러그인과 함께 참조합니다.
> CLAUDE.md에 "디자인 작업 시 docs/디자인가이드.md를 반드시 참조" 추가

---

## 1. 디자인 원칙 (토스에서 배운 3가지)

```
원칙 1: 정보 위계를 시각적으로 확실히 구분한다
  → 가장 중요한 숫자는 크고 굵게
  → 보조 정보는 작고 연한 색으로
  → 같은 영역에 같은 크기의 텍스트가 반복되면 안 됨

원칙 2: 한 화면에서 한 가지만 전달한다
  → 여론조사 탭: "누가 앞서는가"
  → 개요탭: "이 지역은 어떤 곳인가"
  → 후보자탭: "누가 나왔는가"

원칙 3: 여백이 정보를 만든다
  → 빽빽하면 어디를 봐야 할지 모른다
  → 섹션 간 24px, 요소 간 12px, 텍스트 간 8px
  → 카드 안 패딩 16px
```

## 3. 타이포그래피

```css
:root {
  /* 폰트 — 토스는 Toss Product Sans, 우리는 Noto Sans KR */
  --font-family: 'Noto Sans KR', -apple-system, sans-serif;
  
  /* 크기 체계 — 토스식 정보 위계 */
  --text-display: 28px;    /* D-day 카운트다운 */
  --text-headline: 20px;   /* 섹션 제목 */
  --text-title: 16px;      /* 카드 제목, 후보 이름 */
  --text-body: 14px;       /* 본문 */
  --text-caption: 12px;    /* 보조 정보, 날짜 */
  --text-micro: 11px;      /* 뱃지, 태그 */
  
  /* 굵기 — 2가지만 */
  --font-regular: 400;
  --font-bold: 600;
  
  /* 행간 */
  --leading-tight: 1.3;     /* 숫자, 제목 */
  --leading-normal: 1.6;    /* 본문 */
}
```

### 정보 위계 예시

```
[D-day]     --text-display + --font-bold + --text-primary
             → "D-76" 크고 굵고 하양

[섹션 제목]  --text-headline + --font-bold + --text-primary
             → "최신 여론조사" 

[후보 이름]  --text-title + --font-bold + --text-primary
             → "김영록"

[정당명]    --text-body + --font-regular + 정당색
             → "더불어민주당" (파란색)

[지지율]    --text-title + --font-bold + --text-primary
             → "38.2%"

[보조 정보]  --text-caption + --font-regular + --text-tertiary
             → "코리아리서치 · 3/15 · n=1,002"

[뱃지]      --text-micro + --font-bold + 시맨틱색 배경
             → "[접전]" 주황 배경
```

## 4. 간격 시스템

```css
:root {
  /* 4px 단위 (토스 기준) */
  --space-2: 2px;
  --space-4: 4px;
  --space-6: 6px;
  --space-8: 8px;
  --space-12: 12px;
  --space-16: 16px;
  --space-20: 20px;
  --space-24: 24px;
  --space-32: 32px;
  --space-40: 40px;
}
```

### 적용 규칙

```
섹션 간 간격:     --space-24 ~ --space-32
카드 간 간격:     --space-12
카드 내부 패딩:   --space-16
라벨 ↔ 값 간격:  --space-4 ~ --space-8
아이콘 ↔ 텍스트:  --space-6
뱃지 내부 패딩:   --space-4 (상하) --space-8 (좌우)
```

## 6. 영역별 디자인 방향

### 6.1 여론조사 탭

```
핵심: "3초 만에 누가 앞서는지"

구조:
  [한줄요약]   —  가장 큰 텍스트, 뱃지 포함
  [스냅샷 바]  —  정당색 바, 오차범위 해석
  [카드 목록]  —  시간순, 미수집은 접이식

하지 않는 것:
  × Chart.js 그래프 (추이는 나중에)
  × 여러 조사를 합산한 "평균"
  × 무의미한 구분선/테두리 남발
```

### 6.2 개요탭

```
핵심: "이 지역이 어떤 곳인지 30초 이해"

구조:
  [지역 헤더]       — 지역명 + 선거종류 + D-day
  [핵심 숫자 3개]   — 유권자수 / 선거인수 / 투표율(지난)
                      토스 "매트릭 카드" 스타일 (작은 라벨 + 큰 숫자)
  [현직자 정보]     — 이름 + 정당 + 재출마 여부
  [지난 선거 결과]  — 정당색 바 차트 (미니)
  [개요 서술]       — 있으면 표시, 없으면 숨김 (빈 공간 없이)

하지 않는 것:
  × 빈 섹션 표시 (데이터 없으면 섹션 자체를 안 그림)
  × 긴 서술문을 기본으로 펼쳐놓기 (접이식)
```

### 6.3 좌측 사이드바

```
핵심: "선거종류를 직관적으로 선택"

구조:
  [헤더]           — "6.3 지방선거" + D-day (작게)
  [선거종류 목록]  — 아이콘 + 이름 + 숫자 (우측 정렬)
                      활성: 파란 배경 틴트
  [전국 개황]      — 매트릭 카드 4개 (유권자/종류/시도/시군구)
  [정당지지율]     — 미니 바 차트

하지 않는 것:
  × 선거종류 버튼이 너무 크거나 화려
  × 전국 개황에 불필요한 아이콘
  × 스크롤이 필요할 정도로 길게
```

### 6.4 지도 위 UI

```
핵심: "지도를 가리지 않으면서 필요한 컨트롤 제공"

구조:
  [검색]    — 우상단, 둥근 인풋, 포커스 시 넓어짐
  [줌]      — 우하단, 36px 버튼 3개 (확대/축소/초기화)
  [범례]    — 좌하단, 반투명 배경, 최소 크기
  [브레드크럼] — 좌상단, "전국 > 서울 > 종로구" 경로

하지 않는 것:
  × 지도 위에 큰 패널 올리기
  × 범례가 지도 영역을 많이 차지
  × 버튼에 텍스트 라벨 (아이콘만)
```

## 7. Claude Code 지시 방법

### frontend-design 플러그인과 함께 사용

```
"docs/디자인가이드.md를 읽고, 토스 스타일 디자인 시스템에 맞춰
css/style.css의 CSS 변수를 이 가이드의 2~4절로 교체해줘.
기존 컴포넌트 스타일도 5절에 맞춰 수정해줘.
frontend-design 스킬을 활용해서 각 컴포넌트의 완성도를 높여줘."
```

### 영역별 적용

```
1단계: CSS 변수 교체 (컬러 + 타이포 + 간격)
"css/style.css에서 색상, 폰트, 간격 관련 값을
docs/디자인가이드.md의 CSS 변수로 교체해줘.
기존 하드코딩된 색상값도 변수로 바꿔줘."

2단계: 여론조사 탭
"여론조사 탭을 docs/여론조사_실행계획.md + docs/디자인가이드.md에 맞춰
리디자인해줘. 한줄요약 + 스냅샷 바차트 + 카드 구조."

3단계: 개요탭
"개요탭을 토스 매트릭 카드 스타일로 리디자인해줘.
핵심 숫자 3개를 매트릭 카드로, 빈 섹션은 렌더링하지 않도록."

4단계: 사이드바
"좌측 사이드바의 선거종류 버튼과 전국 개황을
docs/디자인가이드.md의 5.5절 스타일로 리디자인해줘."

5단계: 지도 UI
"지도 위 검색, 줌 버튼, 범례를
docs/디자인가이드.md의 5.6절 스타일로 리디자인해줘.
반투명 + 미니멀 + 지도를 가리지 않는 방향으로."
```

## 8. 금지 사항

```
× 그라데이션 배경 (토스는 flat)
× 드롭쉐도우 남발 (깊이는 배경색 차이로)
× 네온/글로우 효과
× 3가지 이상의 폰트 사이즈를 한 카드에
× 아이콘에 색상 (아이콘은 --text-secondary 단색)
× 테두리를 강조 수단으로 사용 (테두리는 구분선, 강조는 배경색 틴트)
× 컴포넌트마다 다른 border-radius (전체 12px 통일, 뱃지만 4px)
× 애니메이션 0.3s 초과 (토스는 0.15s)
```
