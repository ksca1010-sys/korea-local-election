# 6.3 전국지방선거 인터랙티브 선거 정보 지도 - 프로젝트 구조

> 최종 업데이트: 2026-03-19 | 배포: https://korea-local-eletion.pages.dev

---

## 1. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 프론트엔드 | 바닐라 JS + D3.js v7 + Chart.js v4 | 지도·차트·UI |
| 스타일 | CSS3 (반응형) + Font Awesome + Noto Sans KR | 디자인 |
| 데이터 | JSON (정적 호스팅) | Git 추적 가능 |
| 자동화 | Python 3.11 + GitHub Actions | 일일 배치 |
| LLM | Claude Haiku (개요/팩트체크/공약수집) | 자동 분석 |
| API | 네이버 뉴스, NESDC, NEC(선관위) | 외부 데이터 |
| 뉴스 프록시 | Cloudflare Workers | 네이버 뉴스 API CORS 우회 |
| 배포 | Cloudflare Pages | 정적 호스팅 |

## 5. 데이터 흐름

### 5.1 초기화 (App.init)

```
DOMContentLoaded
  → DataLoader: data/static/*.json 11개 병렬 로드 → ElectionData에 merge
  → ElectionData: 후보자/여론조사/역대결과 등 15개 JSON 순차 로드
  → LocalMediaPool + LocalMediaRegistry 로드 → 병합
  → MapModule.init(): TopoJSON 로드 → D3 렌더링
  → ElectionCalendar: 시기별 배너/정렬/공표금지 자동 판정
  → 로딩 화면 제거
```

### 5.2 지역 선택

```
사용자: 지도 클릭 or 검색
  → MapModule: 해당 지역 하이라이트 + 줌
  → App.selectRegion(regionKey, districtName, electionType)
  → 우측 패널 열기
  → 현재 탭 렌더링 (overview/polls/candidates/news/history)
  → 각 탭이 ElectionData에서 해당 지역 데이터 조회
```

### 5.3 뉴스 흐름

```
renderNewsTab(regionKey)
  → 선거유형별 카테고리 빌드 (전체/여론조사/후보·인물/공약·정책)
  → setupLatestNews() → fetchLatestNews()
  → Cloudflare Worker 프록시 → 네이버 뉴스 API
  → evaluateCategoryMatch() (strict → relaxed fallback)
  → 5차원 스코어링 (시의성 25% + 관련성 20% + 신뢰도 10% + 지역성 35% + 인기도 10%)
  → 지역언론 부스트 (+5점, LocalMediaRegistry + LocalMediaPool)
  → 배지 표시 (NEW / 주요 / 지역 / 카테고리)
```

### 5.4 자동화 파이프라인 (백엔드 → 프론트엔드)

```
GitHub Actions (매일)
  │
  ├─ 07:00 KST ─ 선거 개요 업데이트
  │   네이버 뉴스 검색 → Claude Haiku로 narrative 생성
  │   → election_overview.json 업데이트 → git push
  │
  ├─ 09:00 KST ─ 후보자 팩트체크
  │   NEC API + 네이버 뉴스 → Claude로 분석
  │   → governor/superintendent/mayor_candidates.json
  │   → 교차검증 (cross_validate.py)
  │   → git push
  │
  └─ 수시 ─ 여론조사 수집
      NESDC 스크래핑 → PDF 다운로드 → 테이블 파싱
      → polls/polls.json 업데이트 → git push
      │
      ↓
프론트엔드: JSON fetch → UI 렌더링
```

## 6. 전역 객체 API 요약

### ElectionData (data.js)

```javascript
// 정당
ElectionData.getPartyName('democratic')   // '더불어민주당'
ElectionData.getPartyColor('ppp')          // '#E61E2B'

// 지역
ElectionData.regions['seoul']              // { name, population, voters, ... }
ElectionData.getRegion('gyeonggi')         // 광역 정보
ElectionData.getSubRegions('gyeonggi')     // 기초자치단체 목록

// 후보자
ElectionData.getCandidates(regionKey, electionType)
ElectionData.getGovernorCandidates(regionKey)
ElectionData.getMayorCandidates(regionKey, districtName)
ElectionData.getSuperintendentCandidates(regionKey)

// 여론조사
ElectionData.getLatestPollsForDisplay(regionKey, electionType, districtName)
ElectionData.getTrendGroups(regionKey, electionType, districtName)

// 역대 선거
ElectionData.getHistoricalData(regionKey)
ElectionData.getMayorHistoricalData(regionKey, districtName)

// 선거 개요
ElectionData.getElectionOverview(regionKey, electionType, districtName)

// 교육감 성향
ElectionData.getSuperintendentStance(regionKey, candidateName)
ElectionData.getSuperintendentColor(stance)

// 비동기 로드
await ElectionData.loadPollsData()
await ElectionData.loadMayorCandidates()
await ElectionData.loadByElectionData()
// ... 총 15개
```

### MapModule (map.js)

```javascript
MapModule.init()                                    // 초기화
```

## 7. 후보자 데이터 체계

### 상태 분류

| status | 의미 | 판정 기준 |
|--------|------|-----------|
| `DECLARED` | 출마 확정 | 공식 선언 또는 예비후보 등록 |
| `EXPECTED` | 출마 유력 | 복수 언론 보도, 당내 경선 참여 |
| `RUMORED` | 거론 단계 | 단발성 언론 언급, "확실시/유력" 표현 |
| `NOMINATED` | 공천 확정 | 당 공관위/최고위 공식 발표 |
| `WITHDRAWN` | 사퇴/불출마 | 본인 직접 불출마 선언 |

### 데이터 출처 (dataSource)

| dataSource | 의미 |
|-----------|------|
| `incumbent` | 현직 (NEC API 기반) |
| `news` | 뉴스 기반 수집 |
| `news_verified` | Gemini→뉴스 검증 완료 |
| `verified` | 교차검증 통과 |
| `nec_byelection` | NEC 보궐선거 API |
| `claude` | Claude 팩트체크로 추가 |

| 교육감 | `buildSuperintendentNewsCategories()` | 전체/여론조사/후보·인물/교육정책 |
| 재보궐 | (인라인) | 전체/여론조사/후보·인물/공약·정책 |
| 의원 | `buildCouncilNewsCategories()` | 전체/공천·경선/후보·인물 |

### 지역언론 2중 풀 시스템

```
LocalMediaPool (자동 수집)          LocalMediaRegistry (큐레이팅)
 ├─ metro: 광역별 hosts/names       ├─ regions.{key}.province
 └─ municipal: 시군구별 hosts/names  │   ├─ hosts.tier1/tier2
                                     │   ├─ priorityNames
                                     │   └─ outlets[]
                                     └─ regions.{key}.municipalities
                                         └─ {시군구}.hosts/names

→ 앱 초기화 시 Pool을 Registry에 병합 (tier2로)
→ 뉴스 스코어링에서 지역언론 +5점 부스트
→ 개별 뉴스에 "지역" 배지 표시
```
