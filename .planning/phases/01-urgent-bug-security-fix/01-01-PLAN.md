# Plan 01-01: LLM 생성 수치 제거 + UI 방어 처리 + 유령 파일 정리

## Goal
`data.js` 교육감 후보의 LLM 생성 `support` 필드를 전량 제거하고, `overview-tab.js`와 `poll-tab.js`에서 support가 없을 때의 방어 처리를 추가하며, 유령 파일 4개와 stale 주석을 정리한다.

## must_haves
- `data.js:1529-1546` 교육감 candidates 배열의 모든 객체에서 `support` 프로퍼티가 완전히 삭제됨
- `overview-tab.js`에서 `support`가 undefined인 후보에 대해 `여론조사 데이터 없음`이 muted 색상으로 표시됨
- `poll-tab.js`에서 교육감 선택 시 support 있는 후보가 0명이면 여론조사 탭에 `여론조사 데이터 없음` 메시지가 표시됨
- `data.js:1790`의 stale 주석이 정확한 내용으로 수정됨
- 유령 파일 4개(`js/app-state 2.js`, `js/router 2.js`, `js/search 2.js`, `js/sidebar 2.js`)가 삭제됨

### Task 2: overview-tab.js support 없는 후보 방어 처리 (BUG-02, per D-01)

<read_first>
- js/tabs/overview-tab.js (lines 119-128) — `f.latestPoll.results.map(r => ...)` 에서 `r.support`를 사용하여 `${r.name} ${r.support}%` 형태로 렌더링하는 코드
</read_first>

<action>
**Per D-01:** `overview-tab.js` line 123-124에서 `r.support`가 undefined일 때 방어 처리를 추가한다.

현재 코드 (line 123-124):
```javascript
${f.latestPoll.results.map(r =>
    `<span style="font-size:var(--text-caption);background:rgba(255,255,255,0.06);padding:2px 8px;border-radius:4px;">${r.name} ${r.support}%</span>`
).join('')}
```

변경 후:
```javascript
${f.latestPoll.results.map(r =>
    r.support != null
        ? `<span style="font-size:var(--text-caption);background:rgba(255,255,255,0.06);padding:2px 8px;border-radius:4px;">${r.name} ${r.support}%</span>`
        : `<span style="font-size:var(--text-caption);color:var(--text-muted);padding:2px 8px;">${r.name} — 여론조사 데이터 없음</span>`
).join('')}
```

핵심: `r.support != null` (loose equality로 null과 undefined 모두 처리). support가 없는 후보는 `여론조사 데이터 없음`을 `color:var(--text-muted)`로 표시한다.
</action>
