## 1. 현재 사이트의 정보탭 구조 (광역단체장 기준)

```
우측 패널
├── [개요]       — 선거 개요 텍스트, 현직자 정보, 지난 선거 결과
├── [여론조사]   — 여심위 등록 조사 카드, 투명성 안내
├── [후보자]     — 후보자 카드 목록, 공약 비교
├── [뉴스]       — 뉴스 카드 목록 (신뢰도+최신성+관련성 점수 정렬)
└── [역대비교]   — 정권 변화 흐름, 정당 계열 득표율, 역대 선거 결과
```

## 2. 재보궐 정보탭 — 기존과 달라야 하는 점

재보궐은 **국회의원 선거**다. 지방선거와 같은 날 치르지만 성격이 다르다.

| 항목 | 광역단체장/기초단체장 | 재보궐 (국회의원) |
|------|---------------------|-----------------|
| 선거 단위 | 광역시도 / 기초지자체 | 국회의원 선거구 |
| 현직자 | 현 시도지사 / 시군구청장 | **공석** (사퇴/궐위/유죄) |
| "왜 이 선거가 열리는가" | 자연스러운 임기만료 | **사퇴 사유가 핵심 맥락** |
| 역대비교 | 같은 직위 역대 결과 | 해당 선거구 역대 총선 결과 |
| 관련 뉴스 | 지역 중심 | 전국 뉴스 + 지역 뉴스 혼합 |
| 후보 공천 | 당 경선 → 공천확정 | 동일하지만 시기가 다름 |
| 여론조사 | 시도지사/단체장 대상 | 국회의원 선거구 대상 |

**기존 개요탭과 다른 점**:
- "이 선거가 열리는 이유" 섹션이 최상단에 추가
- "현직자 정보" 대신 "전임자 + 사퇴 사유"
- "지난 선거 결과"가 8회 지선이 아니라 **22대 총선** 결과
- 선거구 기본 정보에 관할 행정동 표시 (국회의원 선거구는 읍면동 조합이므로)

**데이터 출처**:
- 전임자 정보: data/candidates/byelection.json (이미 있음)
- 22대 총선 결과: 선관위 공공데이터 (data/prev-elections/에 추가 필요)
- 선거구 기본 정보: 선관위 선거구 획정 데이터

**구현 지시**:
```
"개요탭에서 선거종류가 '재보궐'이면:
1. 최상단에 '이 선거가 열리는 이유' 카드를 표시해줘.
   data/candidates/byelection.json에서 해당 선거구의 
   전임자 이름, 정당, 사퇴 사유를 가져와서 표시.
2. '현직자 정보' 대신 '전임자 정보'로 라벨 변경.
3. '지난 선거 결과'는 8회 지선이 아니라 22대 총선 결과를 표시.
4. 선거구 기본 정보에 관할 행정동 목록을 추가."
```

```
2. 카드 구조는 기존 후보자 카드와 동일.

```

## 4. 좌측 패널 → 중앙 지도 → 우측 정보탭 흐름

```
[좌측] "재보궐" 선택 (5)
    ↓
[중앙] 전국 지도에서 5개 선거구 위치 하이라이트
    ↓ 선거구 클릭 (예: 인천 연수갑)
    ↓
[우측] 재보궐 전용 정보탭 렌더링
       → 개요: "이 선거가 열리는 이유" + 22대 총선 결과
       → 여론조사: 재보궐 전용 필터
       → 후보자: byelection.json에서 로드
       → 뉴스: 재보궐 뉴스 알고리즘 적용
       → 역대비교: 총선 역대 결과
```

**핵심 분기 로직** (app.js에 추가):

```javascript
function renderInfoPanel(region, electionType) {
  if (electionType === "재보궐") {
    // 재보궐 전용 렌더링
    renderByelectionOverview(region);
    // 탭 전환 시에도 재보궐 전용 함수 호출
  } else {
    // 기존 로직 그대로
    renderOverview(region, electionType);
  }
}
```

## 6. Claude Code 실행 순서

```
1단계: 데이터 준비
"data/byelection-districts.json을 만들어줘. 
재보궐 5개 선거구의 기본 정보(유권자 수, 관할 행정동, 사퇴 사유, 전임자)를 넣어줘.
그리고 data/prev-elections/ 에 22대, 21대, 20대 총선의 
해당 5개 선거구 결과 데이터를 추가해줘."

2단계: 개요탭 분기
"app.js에서 선거종류가 '재보궐'일 때 개요탭 렌더링을 분기해줘.
'이 선거가 열리는 이유' 카드를 최상단에, 
22대 총선 결과를 '지난 선거 결과'에 표시."

3단계: 후보자탭 연결
"후보자탭에서 선거종류가 '재보궐'이면 
data/candidates/byelection.json에서 데이터를 가져오도록 해줘."

4단계: 뉴스탭
"docs/재보궐_뉴스_알고리즘_로직.md를 읽고 
뉴스탭에 재보궐 전용 로직을 구현해줘."

5단계: 역대비교탭
"역대비교탭에서 선거종류가 '재보궐'이면 
지방선거 대신 총선 역대 결과를 표시하도록 수정해줘."

6단계: 통합 테스트
"재보궐 5개 선거구를 각각 클릭해서 
5개 탭이 모두 정상 동작하는지 확인해줘."
```
