# 여론조사 파트 완전 재설계

> 문제: 여론조사 데이터가 지역·선거종류에 정확히 분류되지 않아 화면에 안 나오거나 엉뚱한 조사가 나옴
> 목표: 유권자가 "서울시장 여론조사"를 보려 하면 정확히 서울시장 조사만, "인천 연수갑 재보궐"을 보려 하면 정확히 그 조사만 나오는 것
> 이 문서를 docs/ 폴더에 넣고 Claude Code에 "docs/여론조사_완전재설계.md를 읽고 구현해줘" 지시

---

## 1. 현재 문제 진단

### polls.json에 데이터는 있다 (national 248건 + regions 648건)

하지만 화면에 안 나오는 이유:

```
문제 1: 분류가 부정확
  NESDC 원본에는 "여론조사 명칭(지역)"이라는 자유 텍스트 필드만 있음.
  예: "서울특별시 전체 광역단체장선거 정당지지도"
  예: "전라남도 영암군 기초단체장선거 정당지지도"
  예: "전국 정기(정례)조사 정당지지도"
  
  이걸 자동으로 {지역} + {선거종류}로 쪼개야 하는데
  파싱 로직이 부정확하거나 없음.

문제 2: 선거종류 매칭이 안 됨
  사이트에서 "광역단체장"을 선택하고 "서울"을 클릭했을 때
  polls.json에서 electionType="광역단체장" AND region="서울"인 
  데이터를 찾아야 하는데, 이 필터링 키가 안 맞음.

문제 3: 하나의 여론조사에 여러 선거종류가 포함됨
  NESDC 등록 1건에 "광역단체장선거 교육감선거 정당지지도"가 
  같이 들어있는 경우가 많음.
  이걸 각 선거종류별로 분리해서 보여줘야 함.

문제 4: "전국" 조사와 "지역" 조사 구분
  전국 정당지지도 조사는 특정 지역 선거와 다름.
  하지만 전국 조사에도 시도지사 지지율이 포함되는 경우가 있음.
```

## 2. NESDC 데이터 구조 이해

NESDC 등록현황 실제 필드:

```
등록번호: 15449
조사기관명: (주)코리아정보리서치
조사의뢰자: 케이저널
조사방법: 무선 ARS
표본 추출틀: 무선전화번호가상번호
여론조사 명칭(지역): 전라남도 영암군 기초단체장선거 정당지지도
등록일: 2026-02-26
시·도: 전라남도
```

핵심: "여론조사 명칭(지역)" 필드에서 아래 정보를 추출해야 함:
- 지역: 전라남도 영암군
- 선거종류: 기초단체장선거
- 부가정보: 정당지지도

## 3. 분류 알고리즘 — "명칭" 파싱

### 3.1 선거종류 추출

```javascript
function extractElectionTypes(pollTitle) {
  const types = [];
  
  // 정확한 매칭 (순서 중요 — 긴 것부터)
  if (pollTitle.includes("광역단체장선거")) types.push("광역단체장");
  if (pollTitle.includes("기초단체장선거")) types.push("기초단체장");
  if (pollTitle.includes("교육감선거"))     types.push("교육감");
  if (pollTitle.includes("광역의원선거"))   types.push("광역의원");
  if (pollTitle.includes("기초의원선거"))   types.push("기초의원");
  
  // 재보궐은 별도 키워드
  if (pollTitle.includes("재보궐") || pollTitle.includes("보궐선거")) {
    types.push("재보궐");
  }
  
  // 정당지지도만 있고 선거종류가 없는 경우 → "전국 정당지지도" 분류
  if (types.length === 0 && pollTitle.includes("정당지지도")) {
    types.push("정당지지도");
  }
  
  // 정기(정례)조사 → 전국 단위 정기 여론조사
  if (types.length === 0 && pollTitle.includes("정기(정례)조사")) {
    types.push("정기조사");
  }
  
  return types;
}
```

### 3.2 지역 추출

```javascript
function extractRegion(pollTitle, sidoField) {
  // NESDC의 "시·도" 필드가 있으면 우선 사용
  const metro = sidoField; // "전라남도", "서울특별시", "전국" 등
  
  // "전국"이면 전국 단위 조사
  if (metro === "전국" || pollTitle.includes("전국")) {
    return { level: "national", metro: null, municipality: null };
  }
  
  // 기초지자체 추출: "영암군", "수원시", "하남시" 등
  const municipalityPatterns = [
    // 시·군·구 패턴
    /([가-힣]+[시군구])\s/, // "영암군 " "수원시 " 등
    /([가-힣]+[시군구])(?=\s*기초)/, // "영암군기초단체장"
  ];
  
  let municipality = null;
  for (const pattern of municipalityPatterns) {
    const match = pollTitle.match(pattern);
    if (match) {
      // 광역시도명 제거 후 기초지자체명만
      municipality = match[1].replace(metro, "").trim();
      if (municipality) break;
    }
  }
  
  // "전체"가 포함되면 광역시도 전체 조사
  if (pollTitle.includes("전체") || !municipality) {
    return { level: "metro", metro: metro, municipality: null };
  }
  
  return { level: "municipality", metro: metro, municipality: municipality };
}
```

## 4. 사이트 선택 → 여론조사 필터링 매칭

유저가 사이트에서 선택하는 조합과 polls.json 필터링 조건:

```
[유저 선택]                        [polls.json 필터링 조건]

좌측: 광역단체장 + 지도: 서울 →    electionTypes에 "광역단체장" 포함
                                    AND region.metro = "서울특별시"
                                    AND region.level = "metro"

좌측: 교육감 + 지도: 인천 →         electionTypes에 "교육감" 포함
                                    AND region.metro = "인천광역시"

좌측: 기초단체장 + 지도: 영암 →     electionTypes에 "기초단체장" 포함
                                    AND region.municipality = "영암군"

좌측: 재보궐 + 지도: 연수갑 →       electionTypes에 "재보궐" 포함
                                    AND region이 해당 선거구 매칭

좌측: 광역단체장 + 지도: 전국 →     electionTypes에 "광역단체장" 포함
                                    AND region.level = "national"
```

### 핵심 필터 함수

```javascript
function getPollsForSelection(polls, selection) {
  const { electionType, regionKey, municipalityKey } = selection;
  
  return polls.filter(poll => {
    // 1단계: 선거종류 매칭
    const typeMatch = poll.classification.electionTypes.includes(electionType);
    if (!typeMatch) return false;
    
    // 2단계: 지역 매칭
    const region = poll.classification.region;
    
    if (electionType === "광역단체장" || electionType === "교육감") {
      // 광역 선거: 해당 광역시도 조사 + 전국 조사 중 해당 지역 포함
      return region.metro === regionKey || region.level === "national";
    }
    
    if (electionType === "기초단체장") {
      // 기초 선거: 정확한 기초지자체 매칭
      if (region.municipality === municipalityKey) return true;
      // 기초 조사가 없으면 광역 조사에서 해당 지역이 포함된 것
      if (region.metro === regionKey && region.level === "metro") return true;
      return false;
    }
    
    if (electionType === "재보궐") {
      // 재보궐: 선거구 단위 매칭 (별도 로직)
      return matchesByelectionDistrict(poll, municipalityKey);
    }
    
    return false;
  });
}
```

```
+ 오차범위 해석 안내 (격차 vs 오차)
```

### 5.3 오차범위 자동 해석

```javascript
function interpretMarginOfError(results, marginOfError) {
  if (results.length < 2) return null;
  
  const sorted = [...results]
    .filter(r => r.candidateName) // 모름/무응답 제외
    .sort((a, b) => b.support - a.support);
  
  const gap = sorted[0].support - sorted[1].support;
  const doubleMargin = marginOfError * 2; // 양쪽 오차 합산
  
  if (gap > doubleMargin) {
    return {
      type: "outside",
      text: `두 후보의 격차(${gap.toFixed(1)}%p)는 오차범위(±${marginOfError}%p) 밖입니다.`
    };
  } else {
    return {
      type: "within",
      text: `⚠️ 두 후보의 격차(${gap.toFixed(1)}%p)는 오차범위(±${marginOfError}%p) 안이므로 통계적으로 우열을 가릴 수 없습니다.`
    };
  }
}
```

## 6. 추이 차트 — 같은 지역·선거종류의 시간 흐름

### 6.1 동일 기관 추이선 (기본)

```
같은 조사기관이 같은 지역·선거종류를 여러 번 조사했을 때
→ 시간순 라인 차트로 표시
→ "이 기관의 조사에서 추이가 이렇게 변했다"를 보여줌
→ 다른 기관 결과와 섞지 않음 (하우스 이펙트 방지)
```

```javascript
function groupByPollster(polls) {
  const groups = {};
  for (const poll of polls) {
    const key = poll.pollOrg; // 조사기관명
    if (!groups[key]) groups[key] = [];
    groups[key].push(poll);
  }
  // 2건 이상인 기관만 추이선 대상
  return Object.entries(groups)
    .filter(([_, polls]) => polls.length >= 2)
    .map(([org, polls]) => ({
      orgName: org,
      polls: polls.sort((a, b) => 
        new Date(a.surveyDateEnd) - new Date(b.surveyDateEnd)
      )
    }));
}
```

### 6.2 차트 표시 조건

```
조사 0건: "이 지역의 여론조사가 아직 등록되지 않았습니다"
조사 1건: 카드만 표시, 차트 없음
조사 2건+: 카드 + 추이 차트 표시
동일 기관 2건+: 동일 기관 추이선 강조
```

## 7. 선거종류별 여론조사 특수 처리

### 7.1 광역단체장 (시도지사 / 전남광주통합특별시장)

```
- 가장 여론조사가 많은 선거종류
- 전국 조사에도 시도지사 지지율이 포함되는 경우 있음
  → 전국 조사 중 해당 지역 결과만 추출해서 표시
- 정당지지도 조사와 후보 지지율 조사 구분
  → 정당지지도는 별도 섹션으로 표시
```

### 7.2 교육감

```
- 정당 공천이 없어서 후보 인지도가 낮음
- 여론조사 건수 자체가 적음
- "광역단체장선거 교육감선거" 합동 조사가 많음
  → 같은 NESDC 등록 1건에서 교육감 결과만 분리 추출
- 진보/보수/중도 구분이 후보자 카드에 있으면
  여론조사에서도 성향별 색상 적용
```

### 7.3 기초단체장 (구시군장)

```
- 226개 지역 중 여론조사가 있는 곳이 소수
- 대부분 격전지(수원, 하남, 영암 등)만 조사됨
- 여론조사 0건인 지역이 대다수
  → "이 지역의 여론조사가 아직 없습니다.
     {광역시도} 전체 여론조사를 확인하세요." 안내
  → 상위 광역 조사로 폴백하되 구분 명시
```

### 7.4 재보궐 (국회의원)

```
- 5개 지역구뿐이므로 전수 관리 가능
- "재보궐" 키워드로 필터링
- 같은 지역이라도 "인천시장 조사"와 "인천 연수갑 국회의원 조사" 구분 필수
- 전국 재보궐 합산 조사도 있을 수 있음 → 교차 표시
```

## 8. NESDC 크롤러 → polls.json 파이프라인 수정

### 현재 문제
기존 nesdc_scrape.py는 목록 페이지에서 기본 정보만 가져오고
상세 페이지에서 표본수·오차범위·후보별 지지율을 추출하는 로직이 약함.

### 수정 사항

```
[1단계: 목록 수집] — 기존 로직 유지
  NESDC 등록현황 목록 → 등록번호, 조사기관, 의뢰기관, 명칭(지역)

[2단계: 분류 파싱] — 새로 추가
  extractElectionTypes(명칭) → 선거종류 배열
  extractRegion(명칭, 시도필드) → 지역 분류

[3단계: 상세 페이지 스크래핑] — 강화 필요
  각 등록번호의 상세 페이지에서:
  - 표본수
  - 오차범위
  - 조사일시 (시작~종료)
  - 후보별 지지율 결과 (선거종류별로 분리)
  - 정당지지도 결과

[4단계: polls.json 저장]
  위 구조대로 분류 정보 포함하여 저장
```

## 9. 프론트엔드 렌더링 수정

### Claude Code 지시

```
"여론조사 탭 렌더링을 수정해줘:

1. 유저가 선택한 {선거종류} + {지역}에 정확히 매칭되는 
   여론조사만 필터링해서 보여줘.
   getPollsForSelection 함수를 이 문서의 로직대로 구현.

2. 각 여론조사 카드에 7가지 필수 정보를 표시:
   조사기관, 의뢰기관, 조사일, 조사방법, 표본수, 오차범위, 지지율.

3. 오차범위 자동 해석을 카드 하단에 표시.
   격차가 오차범위 안이면 '통계적으로 우열을 가릴 수 없습니다' 안내.

4. 같은 기관의 조사가 2건 이상이면 추이 차트를 표시.

5. 해당 지역의 여론조사가 0건이면:
   - 기초단체장: '이 지역의 여론조사가 아직 없습니다. {광역} 전체를 확인하세요.'
   - 교육감: '교육감 여론조사가 아직 없습니다.'
   - 재보궐: '이 선거구의 여론조사가 아직 없습니다.'

6. 하나의 NESDC 등록에 여러 선거종류가 포함된 경우
   (예: 광역단체장 + 교육감) 해당 선거종류 결과만 추출해서 표시."
```

## 10. 실행 순서 요약

```
1단계: nesdc_scrape.py 분류 파싱 로직 추가 → 크롤러 실행
       → polls.json에 classification 필드가 채워진 데이터 저장

2단계: getPollsForSelection 필터 함수 구현
       → 선거종류 + 지역 조합으로 정확 필터링

3단계: 여론조사 카드 렌더링 수정
       → 7가지 필수 정보 + 오차범위 해석 표시

4단계: 추이 차트 구현
       → 동일 기관 추이선

5단계: 0건 처리 + 폴백 로직
       → 기초지역은 광역 폴백, 안내 문구

6단계: 전남광주통합특별시 매핑
       → 광주 + 전남 → 통합특별시로 자동 변환
```
