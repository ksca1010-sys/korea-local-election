## 2. 선거종류별 화면흐름

### 2.1 광역단체장 (governor)

```text
[필터: 광역단체장] → 전국 지도 (정당 색상)
    │
    ├─ 광역 클릭 (예: 경기도)
    │   └─ 우측 패널 열림
    │       ├─ 개요탭: LLM 개요 + 현직자 정보 + 지역 이슈
    │       ├─ 여론조사탭: 통합추세 + 추이차트 + 카드목록
    │       ├─ 후보자탭: 후보 카드 (정당·경력·공약)
    │       ├─ 뉴스탭: [전체][여론조사][후보·인물][공약·정책]
    │       └─ 역대비교탭: 1~8회 역대 결과
    │
    └─ 시군구 클릭 → 기초단체장으로 drill-down
```

### 2.2 교육감 (superintendent)

```text
[필터: 교육감] → 전국 지도 (성향 색상: 진보/보수/중도)
    │
    └─ 광역 클릭 (예: 서울)
        └─ 우측 패널
            ├─ 개요탭: LLM 개요 + 현직 교육감 + 교육 지표
            ├─ 여론조사탭: 교육감 여론조사
            ├─ 후보자탭: 성향별 후보 카드
            ├─ 뉴스탭: [전체][여론조사][후보·인물][교육정책]
            └─ 역대비교탭: 교육감 역대 결과
```

### 2.3 기초단체장 (mayor)

```text
[필터: 기초단체장] → 전국 지도
    │
    ├─ 광역 클릭 → 시군구 지도로 전환 + 안내 메시지
    │
    └─ 시군구 클릭 (예: 종로구)
        └─ 우측 패널
            ├─ 개요탭: LLM 개요 + 현직 단체장
            ├─ 여론조사탭: 해당 시군구 여론조사
            ├─ 후보자탭: 후보 카드 (공약 포함)
            ├─ 뉴스탭: [전체][여론조사][후보·인물][공약·정책]
            └─ 역대비교탭: 기초단체장 역대 결과
```

### 2.4 광역의원 (council)

```text
[필터: 광역의원] → 전국 지도
    │
    ├─ 광역 클릭 → 광역의원 선거구 지도 (정당 색상)
    │   └─ 안내: "시군구를 클릭하면 선거구를 확인할 수 있습니다"
    │
    ├─ 시군구 클릭 → 해당 시군구 선거구 확대
    │   └─ 선거구 칩 버튼 목록
    │
    └─ 선거구 클릭 (예: 종로구제1선거구)
        └─ 우측 패널
            ├─ 개요탭: 선거구 정보 + 관할 읍면동
            │   (LLM 개요 없음)
            ├─ 현직자 정보: 현직 의원 카드 + 8회 지선 결과
            ├─ 여론조사탭: 안내 (의원급 여론조사 거의 없음)
            ├─ 후보자탭: 현직 기반 후보 카드
            ├─ 뉴스탭: [전체][공천·경선][후보·인물]
            └─ 역대비교탭: 3~8회 역대 결과 (당선+차점자)
```

### 2.5 기초의원 (localCouncil)

```text
[필터: 기초의원] → 전국 지도
    │
    ├─ 광역 클릭 → 시군구 지도로 전환 + 안내 메시지
    │
    ├─ 시군구 클릭 (예: 종로구)
    │   └─ 우측 패널
    │       └─ 선거구 카드 목록 (클릭 가능)
    │           각 카드: 선거구명 + 의석수 + 후보 미리보기
    │
    └─ 선거구 클릭 (예: 종로구가선거구)
        └─ 우측 패널
            ├─ 개요탭: 선거구 정보 + 관할 읍면동
            ├─ 현직자 정보: 현직 의원 + 8회 지선 결과
            ├─ 여론조사탭: 안내 (의원급 여론조사 없음)
            ├─ 후보자탭: 중대선거구 → 정당별 그룹핑
            │            소선거구 → 카드 나열
            ├─ 뉴스탭: [전체][공천·경선][후보·인물]
            └─ 역대비교탭: 8회 결과
```

### 2.6 광역비례 (councilProportional)

```text
[필터: 광역비례] → 전국 지도 (도넛 오버레이)
    │
    └─ 광역 클릭 (예: 전남)
        └─ 우측 패널
            ├─ 개요탭: 비례대표 설명 + 배분 의석 수
            ├─ 현직자 정보: 정당별 현직 비례 의원 이름
            ├─ 후보자탭: 5/15 전 안내 / 후 명부 테이블
            ├─ 뉴스탭: 해당 지역 뉴스
            └─ 역대비교탭: 5~8회 득표율 바차트 + 의석 배분
            (여론조사탭 없음)
```

### 2.7 기초비례 (localCouncilProportional)

```text
[필터: 기초비례] → 전국 지도
    │
    ├─ 광역 클릭 → 시군구 지도 (의석수 라벨) + welcome 유지
    │   (패널 안 열림)
    │
    └─ 시군구 클릭 (예: 홍성군)
        └─ 우측 패널
            ├─ 개요탭: 기초의원 비례대표 설명 + 배분 의석
            ├─ 현직자 정보: 정당별 현직 비례 의원 이름
            ├─ 후보자탭: 5/15 전 안내 / 후 명부 테이블
            ├─ 뉴스탭: 해당 지역 뉴스
            └─ 역대비교탭: 4~8회 득표율 + 의석 배분
            (여론조사탭 없음)
```

### 2.8 재보궐 (byElection)

```text
        └─ 우측 패널
            ├─ 개요탭: 재보궐 사유 + 이전 당선자
            ├─ 여론조사탭: 해당 선거구 여론조사
            ├─ 후보자탭: 후보 카드
            ├─ 뉴스탭: [전체][여론조사][후보·인물][공약·정책]
```

## 5. 뉴스 시스템 흐름

```text
사용자: 뉴스탭 클릭
    │
    ├─ 카테고리 선택 (전체/여론조사/후보·인물/공약·정책)
    │
    ├─ fetchLatestNews()
    │   └─ Cloudflare Worker 프록시 → 네이버 뉴스 API
    │       ├─ 주 쿼리 + 대체 쿼리 (altQueries)
    │       └─ sort=date + sort=sim (인기순)
    │
    ├─ evaluateCategoryMatch() — strict → relaxed 폴백
    │   ├─ mustAny / targetAny / excludeAny 규칙
    │   └─ boostHosts (지역언론 +5점)
    │
    ├─ 5차원 스코어링
    │   ├─ 시의성 25% (최신 = 높음)
    │   ├─ 관련성 20% (키워드 매칭)
    │   ├─ 신뢰도 10% (주요언론 1.0, 지역1티어 0.85)
    │   ├─ 지역성 35% (지역언론 = 1.0)
    │   └─ 인기도 10% (네이버 순위)
    │
    ├─ 배지 표시: NEW / 주요 / 지역 / 카테고리
    │
    └─ 렌더링: 카드 목록 (6개씩, 더보기)
```

## 6. 여론조사 시스템 흐름

```text
사용자: 여론조사탭 클릭
    │
    ├─ 공표금지 체크 (5/28~6/3 18:00)
    │   └─ 금지 시: "선거법에 따라 표시 불가" 안내
    │
    ├─ ① 통합 추세 요약 (가중 이동평균)
    │   ├─ 최근 21일 조사 가중평균
    │   ├─ 최신성 × 표본크기 가중치
    │   └─ 접전 여부 판정 (격차 < 오차범위×2)
    │
    ├─ ② 추이 차트 (동일 기관 2회+)
    │   └─ Chart.js 라인 차트
    │
    ├─ ③ 돌출 조사 감지
    │   ├─ 1위-2위 격차 기준 2σ 이상
    │   └─ 해당 카드에 주황 경고
    │
    └─ ④ 전체 카드 목록
        ├─ 조사기관 + 방법 배지 (ARS/전화면접)
        ├─ 후보별 바차트 + 득표율
        ├─ 오차범위 해석 (접전 판정)
        └─ 여심위 원본 링크
```

## 7. 선거종류 전환 흐름

```text
사용자: 선거종류 필터 클릭
    │
    ├─ getElectionUnit() 비교
    │   ├─ 같은 단위 → districtName 유지
    │   └─ 다른 단위 → districtName 초기화
    │       metro: 광역단체장, 교육감, 광역비례
    │       district: 기초단체장, 기초비례
    │       constituency: 광역의원, 기초의원, 재보궐
    │
    ├─ MapModule.setElectionType(newType)
    │   ├─ 지도 색상 변경
    │   ├─ province 모드로 복귀
    │   └─ 특수 처리: 재보궐 마커, 비례 도넛
    │
    └─ 패널 리셋 (welcome 화면)
```

## 8. 지도 상호작용

```text
[Province 모드] 전국 17개 광역
    │
    ├─ 클릭 → selectRegion()
    │   ├─ governor/superintendent: 패널 열림
    │   ├─ mayor/localCouncil: 시군구 지도 전환
    │   ├─ council: 광역의원 선거구 지도 전환
    │   ├─ localCouncilProportional: 시군구 지도 (패널 안 열림)
    │   └─ councilProportional: 패널 열림
    │
    ├─ 호버 → 지역명 + 유권자수 툴팁
    └─ 스크롤 → 줌 인/아웃

[District 모드] 시군구 경계
    │
    ├─ 클릭 → onDistrictSelected()
    │   ├─ mayor: 기초단체장 상세
    │   ├─ localCouncil: 기초의원 선거구 목록
    │   └─ localCouncilProportional: 기초비례 상세
    │
    └─ 뒤로가기 → province 모드

[Constituency 모드] 선거구 경계 (광역의원/기초의원)
    │
    ├─ 클릭 → onConstituencySelected()
    │   └─ CouncilTab 렌더링
    │
    └─ 뒤로가기 → district 모드
```

## 10. 데이터 로딩 UX

```text
초기 로딩:
    ├─ 로딩 화면 (스피너 + 진행률)
    ├─ DataLoader: 11개 정적 JSON 병렬 로드

    └─ MapModule.init(): TopoJSON 로드 → D3 렌더링

탭 전환:
    ├─ 스켈레톤 로딩 UI (뉴스/여론조사)
    ├─ 비동기 데이터 → 로드 완료 후 렌더
    └─ 캐시: sessionStorage (뉴스 10분, 에러 시 30분)
```
