## 1. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 프론트엔드 | 바닐라 JS + D3.js v7 + Chart.js v4 | 지도·차트·UI |
| 스타일 | CSS3 (반응형) + Font Awesome + Noto Sans KR | 디자인 |
| 데이터 | JSON (정적 호스팅) | Git 추적 가능 |
| 자동화 | Python 3.11 + GitHub Actions | 일일 배치 |
| LLM | Claude Haiku (개요/팩트체크/공약수집) | 자동 분석 |
| API | 네이버 뉴스, NESDC, NEC(선관위) | 외부 데이터 |
| 뉴스 프록시 | Cloudflare Workers | 네이버 뉴스 API CORS 우회 |
| 배포 | Cloudflare Pages | 정적 호스팅 |

## 6. 뉴스 시스템 아키텍처

### 프록시
```
프론트엔드 → Cloudflare Worker → 네이버 뉴스 API
              https://election-news-proxy.ksca1010.workers.dev
```

### 5차원 스코어링
```
점수 = (시의성 × 25%) + (관련성 × 20%) + (신뢰도 × 10%) + (지역성 × 35%) + (인기도 × 10%)
```

### 지역언론 2중 풀
```
LocalMediaPool (자동 수집) + LocalMediaRegistry (큐레이팅)
→ 앱 초기화 시 Pool을 Registry에 병합
→ 지역언론 기사 +5점 부스트 + "지역" 배지
```

## 7. 후보자 상태/출처 체계

### 상태 (status)
| 값 | 의미 | 판정 기준 |
|----|------|----------|
| `DECLARED` | 출마 확정 | 공식 선언 또는 예비후보 등록 |
| `EXPECTED` | 출마 유력 | 복수 언론 보도 |
| `RUMORED` | 거론 단계 | "확실시/유력" 표현 |
| `NOMINATED` | 공천 확정 | 당 공관위 공식 발표 |
| `WITHDRAWN` | 사퇴/불출마 | 본인 직접 선언 |

### 출처 (dataSource)
| 값 | 의미 |
|----|------|
| `incumbent` | 현직 (NEC API) |
| `news` | 뉴스 기반 수집 |
| `news_verified` | 뉴스 검증 완료 |
| `verified` | 교차검증 통과 |
| `claude` | Claude 팩트체크 추가 |
