# Requirements: 선거정보지도 v1.1

**Defined:** 2026-03-29
**Core Value:** 허위 데이터 없이, 모든 선거구의 후보·여론조사·역대 결과를 한 화면에서 빠르게 탐색

---

## v1.1 Requirements (Active)

### 여론조사 파이프라인 (POLL)

- [ ] **POLL-01**: 사용자가 여론조사 빈값(지지율 누락) 15건을 채운 최신 데이터를 볼 수 있다
- [ ] **POLL-02**: 선거일까지 신규 여론조사를 파이프라인으로 지속 추가할 수 있다 (수동 재실행 가능)

### 본후보 등록 대응 (CAND)

- [ ] **CAND-01**: 본후보 등록 후 candidates 데이터가 공식 후보(NOMINATED)로 자동 전환된다
- [ ] **CAND-02**: 후보 기호가 배정되면 기호순 정렬이 올바르게 적용된다 (5/15 18:00 이후)
- [ ] **CAND-03**: 등록 취소·무효 후보가 목록에서 제거된다

### 개표 시스템 완성 (ELEC)

- [ ] **ELEC-01**: NEC 개표 API URL을 캡처하여 Worker에 기입하는 절차가 문서화된다
- [ ] **ELEC-02**: Worker 통합 테스트가 완료되어 실제 응답을 검증한다
- [ ] **ELEC-03**: 브라우저 UAT 3건(지도 시각화·폴백 UI·개표 배너)이 완료된다

### 선거일 운영 준비 (OPS)

- [ ] **OPS-01**: 선거일 배포 체크리스트가 존재하고 최종 배포가 완료된다
- [ ] **OPS-02**: 공표금지 기간(5/28~6/3) 여론조사 자동 숨김이 검증된다
- [ ] **OPS-03**: Worker 장애 시 수동 폴백(JSON) 전환 절차가 검증된다

---

## Future Requirements (v1.2+)

- 광역의원/기초의원 후보 데이터 (본후보 등록 이후 선관위 API 수집)
- 선거 결과 아카이브 (6/3 개표 완료 후 영구 보존)
- 모바일 UX 개선 (후보 검색 강화)

## Out of Scope (v1.1)

- 사용자 계정/로그인 — 정보 조회 전용 서비스
- 서버사이드 렌더링/SPA 프레임워크 전환
- AI 분석 수치 — 허위 데이터 방지 원칙 (헌법 제2조)
- 자동화 테스트 인프라 — 바닐라 JS 수동 검증 유지
