## 아키텍처 요약

- **서버사이드 자동화**: GitHub Actions 후보자 크론 1개만 존재
- **가장 복잡한 알고리즘**: 여론조사 6단계 필터 파이프라인 (`data.js:getPollsByRegion`)
- **가장 정교한 NLP**: YAKE 기반 쟁점 추출 엔진 (`issue_engine.js`)
- **데이터 흐름**: NESDC/NEC → Python 파이프라인 → JSON → 프론트엔드 fetch → 클라이언트 가공 → D3/Chart.js 시각화
