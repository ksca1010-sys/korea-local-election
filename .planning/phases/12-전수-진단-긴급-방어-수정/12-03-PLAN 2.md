**Step 1: data-health-check.yml actions:write 필요성 확인**

`scripts/data_health_check.py`를 읽어 `gh run rerun` 또는 `gh api` 등 actions:write를 필요로 하는 호출이 실제로 있는지 확인한다.

- 있으면: actions:write를 유지하고 주석으로 근거를 명시
- 없으면: `actions: read`로 다운그레이드

RESEARCH.md에 따르면 "data_health_check.py가 GH_TOKEN으로 gh run rerun을 호출하여 실패한 워크플로우를 자동 재실행하는 기능이 있다"고 분석되었으므로, 실제 코드에서 해당 부분을 찾아 확인한다.

**Step 2: update-gallup.yml trigger-overview job permissions 확인**

trigger-overview job의 `actions: write`는 `gh workflow run update-overview.yml`을 호출하기 위해 필요하다. job-level 선언이 올바른지 확인하고 주석을 추가한다.

**Step 3: 주석 정규화 적용**

두 파일의 permissions 블록에 각 권한의 필요 근거를 한국어 주석으로 명시한다:

```yaml
# data-health-check.yml 예시:
permissions:
  contents: write   # git push — 헬스체크 결과 data/ 파일 커밋
  actions: write    # gh run rerun — 실패 워크플로우 자동 재실행 (heal_state)

# update-gallup.yml trigger-overview job 예시:
permissions:
  actions: write    # gh workflow run — update-overview.yml 트리거
```

다른 13개 워크플로우는 이미 `contents: write`만 선언되어 있어 최소 권한 원칙에 부합하므로, 이 2개 파일만 주석을 추가한다. 나머지 워크플로우에 주석을 추가하는 것은 범위 밖이다(변경 최소화).
