# 도넛 차트 디자인 검증

생성일: 2026-03-09
모델: gemini-2.5-flash

---

안녕하세요! 데이터 시각화 및 웹 UI/UX 디자인 전문가로서 한국 지방선거 인터랙티브 지도 프로젝트의 도넛 차트 문제에 대해 자세히 분석하고 실용적인 해결책을 제안해 드리겠습니다.

현재 지도 위에 도넛 차트를 직접 표시하는 방식은 작은 공간에서 여러 정당의 비율을 비교하기 어렵고, 특히 마우스 오버 시 애니메이션은 사용 경험을 저해하는 요인으로 보입니다.

---

## 1. 도넛 차트 대안 디자인 제안

지도 위 도넛 차트를 대체할 수 있는, 더 가독성 높은 시각화 방식 3가지를 다크 테마(#0a0e17)에서의 가독성을 고려하여 제안합니다.

### 대안 1: 컴팩트 스택 막대 차트 (Compact Stacked Bar Chart)

각 지역 경계 내부에 정당별 의석 비율을 나타내는 작은 가로 또는 세로 스택 막대 차트를 배치합니다. 도넛 차트와 동일하게 "전체 대비 부분"의 비율을 표현하지만, 원형이 아닌 선형이므로 길이 비교를 통해 비율을 더 직관적으로 파악할 수 있습니다.

*   **장점:**
    *   **가독성:** 길이 비교는 원형의 각도/면적 비교보다 인간의 인지 능력이 더 우수하여 비율 차이를 명확하게 파악할 수 있습니다.
    *   **공간 효율성:** 특히 가로 막대 차트의 경우 지도 내 지역 라벨과 겹치지 않도록 배치하기 용이합니다.
    *   **다크 테마:** 배경과 대비되는 선명한 색상 팔레트 사용 시 각 세그먼트가 뚜렷하게 보입니다.
*   **단점:**
    *   **공간 제약:** 지역이 너무 작거나 모양이 불규칙하면 여전히 배치 및 라벨링에 어려움이 있을 수 있습니다.
    *   **작은 세그먼트 문제:** 매우 작은 비율의 세그먼트는 여전히 보이기 어려울 수 있습니다.
*   **다크 테마 고려:**
    *   막대 내부 색상은 `#0a0e17` 배경과 강하게 대비되면서도 인접한 색상과 구별되는 명도/채도 차이를 가진 색상을 사용해야 합니다. (예: 밝은 파랑, 밝은 녹색, 주황 등)
    *   막대 테두리는 매우 얇은 회색(예: `#444`)으로 설정하여 인접한 막대 차트와의 구분을 돕습니다.

### 대안 2: 선두 정당 기반 코로플레스 맵 + 상세 정보 툴팁 (Leader-Highlight Choropleth Map + Detailed Tooltip)

각 지역을 해당 지역에서 가장 많은 의석을 차지한 정당의 색상으로 칠하고, 마우스 오버(Hover) 또는 클릭 시 해당 지역의 정당별 상세 의석 비율 정보를 툴팁(Tooltip)으로 제공하는 방식입니다.

*   **장점:**
    *   **지도 가독성 극대화:** 지도가 매우 깔끔해지며, 사용자는 전체적인 판세(어느 정당이 어느 지역에서 강세인지)를 한눈에 파악하기 매우 용이합니다. "한눈에 정당 분포 파악"이라는 목표에 가장 부합합니다.
    *   **정보의 점진적 공개:** 초기에는 큰 그림을 보여주고, 사용자가 필요할 때만 상세 정보를 제공하여 정보 과부하를 줄입니다.
    *   **다크 테마:** 각 정당의 대표 색상을 채워 넣기만 하면 되므로, 색상 팔레트 관리 및 가독성 확보가 용이합니다.
*   **단점:**
    *   **상세 정보 즉시 파악 불가:** 특정 지역의 모든 정당 의석 비율을 보려면 반드시 인터랙션(마우스 오버/클릭)이 필요합니다.
    *   **툴팁 구현 난이도:** 툴팁 내부에 복잡한 시각화(예: 다시 도넛 차트)를 넣으려면 추가적인 구현 작업이 필요합니다.
*   **다크 테마 고려:**
    *   정당별 색상은 `#0a0e17` 배경 위에서 서로 명확히 구분되며, 지도 경계선은 흰색 또는 밝은 회색(예: `#bbb`)으로 얇게 처리하여 지역 구분을 명확히 합니다.
    *   툴팁 배경은 어두운 색상(예: `#1a202c`)으로 설정하고, 텍스트는 밝은 색상(예: `#eee`, `#fff`)을 사용하여 높은 대비를 유지합니다. 툴팁 내부에 표시되는 미니 차트나 정당별 색상 표시도 배경과 잘 어울리게 디자인합니다.

### 대안 3: 미니 차트형 텍스트 박스 (Mini Chart-like Text Box with Simplified Summary)

각 지역의 주요 정당 1~2개의 이름과 의석 비율(%)을 텍스트로 직접 표시하고, 나머지 정당은 "기타" 등으로 묶거나 마우스 오버 시 상세 정보를 보여주는 방식입니다. 이 때 텍스트 박스 배경에 해당 정당의 색상을 옅게 깔아주거나, 작은 바 차트 요소를 추가하여 시각적 강조를 더할 수 있습니다.

*   **장점:**
    *   **정확한 정보 전달:** 텍스트 기반이므로 정확한 숫자 정보를 전달할 수 있습니다.
    *   **간결성:** 복잡한 차트 대신 최소한의 정보만 표시하여 지도의 복잡성을 줄일 수 있습니다.
    *   **다크 테마:** 텍스트 색상과 배경색 대비만 잘 맞추면 되므로 구현이 비교적 간단합니다.
*   **단점:**
    *   **시각적 비교의 한계:** 숫자만으로는 직관적인 비율 비교가 어렵습니다 (특히 막대/도넛 차트 대비).
    *   **공간 제약:** 여러 정당을 표시해야 할 경우 여전히 공간 문제에 부딪힐 수 있습니다. 지역명과 겹치지 않도록 조절이 필요합니다.
*   **다크 테마 고려:**
    *   텍스트 색상은 `#0a0e17` 배경 위에서 눈에 잘 띄는 밝은 색(예: 흰색, 밝은 회색)을 사용합니다.
    *   텍스트 박스의 배경색은 정당 색상 중 하나를 채도가 낮은 형태로 사용하거나, 어두운 회색(예: `#222`)으로 설정하여 텍스트와 명확한 대비를 이룹니다.

## 2. 도넛 차트를 유지할 경우 개선안

현재 도넛 차트를 유지하기로 결정하셨다면, 가독성 향상과 애니메이션 문제 해결을 위한 구체적인 Chart.js 설정값을 제안합니다.

### 2.1. Hover 애니메이션 제거/축소 방법

가장 큰 문제는 `hover` 시 세그먼트가 확대되는 애니메이션이므로, 이를 완전히 제거하거나 최소화해야 합니다. `Chart.js`의 `options` 객체 내에서 설정을 변경합니다.

```javascript
// charts.js 파일 또는 도넛 차트 생성 시 전달하는 options 객체에 추가/수정
options: {
    // 1. 전역 애니메이션 비활성화 (Hover 효과 포함)
    animation: false, // 모든 애니메이션을 비활성화 (가장 강력한 방법)

    // 만약 다른 곳에 애니메이션이 필요하다면, hover 애니메이션만 제어
    // hover: {
    //     animationDuration: 0, // hover 시 애니메이션 지속 시간을 0으로 설정
    //     mode: 'nearest',      // hover 상호작용 모드. 'none'으로 설정하면 아무런 hover 효과도 발생하지 않음.
    //     intersect: false      // 마우스가 요소 위에 정확히 있어야 hover 되는지 여부. false면 가까이만 가도 동작.
    // },
    
    // interaction: {
    //     mode: 'none' // 모든 사용자 인터랙션 (hover, click)을 비활성화
    // },

    // 2. 툴팁 관련 설정 (툴팁이 춤추는 원인이 될 수도 있음)
    plugins: {
        tooltip: {
            enabled: true, // 툴팁 활성화 여부 (원치 않으면 false)
            animation: false, // 툴팁 자체 애니메이션 비활성화
            // hover 시 세그먼트가 확대되는 것이 툴팁이 활성화될 때 발생하는 부수 효과일 수 있으므로
            // 툴팁의 인터랙션 모드도 조절할 수 있습니다.
            // interaction: {
            //     mode: 'nearest',
            //     intersect: false
            // }
        },
        legend: {
            display: false // 범례는 지도 위 공간 절약을 위해 비활성화 권장
        },
        // Chart.js 3.x 이상에서 datalabels 플러그인을 사용한다면, 해당 플러그인 설정도 확인
        // datalabels: {
        //     animation: { duration: 0 }
        // }
    },
    
    // elements: {
    //     arc: {
    //         // hover 스타일을 일반 스타일과 동일하게 설정하여 변화 없애기
    //         hoverBorderWidth: 0, // default 0
    //         hoverBorderColor: 'rgba(0,0,0,0)',
    //         hoverBackgroundColor: null // 기본 색상 유지
    //     }
    // }

    // 반응형 비활성화 (맵 위의 작은 차트는 고정 크기가 유리할 수 있음)
    responsive: false,
    maintainAspectRatio: false
}
```
**핵심 개선:** `animation: false` 또는 `hover: { animationDuration: 0 }`, `interaction: { mode: 'none' }` 으로 설정하여 애니메이션을 완전히 제거하는 것이 가장 확실합니다.

### 2.2. 가독성 향상을 위한 구체적인 Chart.js 설정값

#### 1) 최소 크기 및 도넛 두께 조절 (`cutout`)

지도 위 차트의 크기는 제한적이므로, 도넛의 두께를 조절하여 각 세그먼트의 면적을 최적화해야 합니다.
`cutout` 값은 도넛 가운데 뚫린 원의 크기를 조절합니다. 이 값이 작을수록 도넛의 면적이 커져 세그먼트가 더 잘 보입니다.

```javascript
options: {
    // ... (위의 애니메이션 설정)
    cutout: '60%', // 도넛의 중앙 구멍 크기를 60%로 설정 (기본 50%보다 면적 증가)
                  // '70%'는 더 얇은 도넛이 되고, '50%'는 더 두꺼운 도넛이 됩니다.
                  // 지도 위 작은 차트는 60% 또는 50% 정도가 적당합니다.
    elements: {
        arc: {
            borderWidth: 1, // 각 세그먼트의 경계선 두께
            borderColor: '#333' // 각 세그먼트의 경계선 색상 (다크 테마에 어울리는 어두운 회색)
        }
    }
}
```

#### 2) 색상 대비

다크 테마(#0a0e17) 환경에서 각 정당의 색상이 명확하게 구분되어야 합니다.

*   **색상 팔레트:** `#0a0e17` 배경 위에서 시각적으로 잘 구분되는 고채도의 밝은 색상 팔레트를 사용합니다. 정당 수가 많다면 유사한 색상은 피하고, 필요하다면 소수 정당을 "기타"로 묶는 것도 고려합니다.
    *   **예시 (다크 테마용):**
        *   `#FF6384` (밝은 빨강)
        *   `#36A2EB` (밝은 파랑)
        *   `#FFCE56` (밝은 노랑)
        *   `#4BC0C0` (민트)
        *   `#9966FF` (밝은 보라)
        *   `#FF9F40` (밝은 주황)
        *   `#C9CBCF` (밝은 회색 - 기타)
*   **경계선:** 각 세그먼트의 `borderColor`를 설정하여 인접한 세그먼트 간의 구분을 명확히 합니다. (`#333` 또는 `#444`와 같이 어두운 배경에 살짝 보이는 색상)

#### 3) 라벨 표시 방식 (Chart.js Datalabels Plugin 사용 권장)

기본 `Chart.js`는 도넛 차트 내부에 직접적인 라벨링을 지원하지 않습니다. `chartjs-plugin-datalabels` 플러그먼트를 사용하면 각 세그먼트 내부에 백분율 또는 정당명을 표시할 수 있습니다.

```javascript
// charts.js 파일 내 차트 초기화 시
// 먼저, chartjs-plugin-datalabels 플러그인을 설치하고 임포트해야 합니다.
// import ChartDataLabels from 'chartjs-plugin-datalabels';
// Chart.register(ChartDataLabels);

options: {
    // ...
    plugins: {
        legend: {
            display: false // 지도 위 공간 절약을 위해 범례는 비활성화
        },
        tooltip: {
            enabled: true, // 툴팁은 상세 정보 제공을 위해 활성화
            backgroundColor: '#1a202c', // 다크 테마 툴팁 배경색
            titleColor: '#fff', // 툴팁 제목 색상
            bodyColor: '#eee', // 툴팁 내용 색상
            borderColor: '#555', // 툴팁 경계선 색상
            borderWidth: 1,
            displayColors: true // 툴팁 내 색상 박스 표시
        },
        datalabels: { // chartjs-plugin-datalabels 설정
            display: 'auto', // 세그먼트 크기에 따라 자동으로 표시 여부 결정
            color: '#fff',   // 라벨 텍스트 색상 (밝은 색으로 대비)
            font: {
                weight: 'bold',
                size: 8 // 작은 차트이므로 폰트 크기를 작게 설정
            },
            formatter: (value, context) => {
                // 데이터 값(의석 수)을 백분율로 변환하여 표시
                const total = context.chart.data.datasets[0].data.reduce((a, b) => a + b, 0);
                const percentage = ((value / total) * 100).toFixed(1);
                // 작은 세그먼트는 라벨이 겹치거나 가독성이 떨어지므로 특정 비율 이하는 표시하지 않음
                return percentage > 5 ? percentage + '%' : ''; // 5% 미만은 표시 안 함
                // 또는 `context.chart.data.labels[context.dataIndex]` 로 정당명 표시 가능
            },
            anchor: 'center', // 라벨 위치 (가운데)
            align: 'center',  // 라벨 정렬
            // textShadowColor: '#000', // 텍스트 그림자로 가독성 향상 (옵션)
            // textShadowBlur: 2
        }
    }
}
```

## 3. 권장안

위에서 제안한 방안 중, 현재 프로젝트의 목표("사용자가 한눈에 정당 분포를 파악할 수 있어야 합니다.")와 지도라는 제약 사항을 고려했을 때, **'선두 정당 기반 코로플레스 맵 + 상세 정보 툴팁'** 방식을 가장 강력하게 권장합니다.

### 권장안: 선두 정당 기반 코로플레스 맵 + 상세 정보 툴팁

**선택 이유:**

1.  **최고의 지도 가독성:** 지도는 지역적인 패턴과 트렌드를 보여주는 데 최적화되어야 합니다. 각 지역에 복잡한 차트를 올리는 것은 지도를 지저분하게 만들고, 전체적인 흐름을 방해합니다. 코로플레스 맵은 지도를 가장 깔끔하게 유지하면서도 각 지역의 '승리' 또는 '강세' 정당을 직관적으로 보여주어 전체적인 판세를 한눈에 파악하기에 가장 좋습니다.
2.  **정보의 점진적 공개:** 사용자는 처음에는 전체적인 흐름을 파악하고, 특정 지역에 대한 상세 정보가 궁금할 때만 마우스를 올리거나 클릭하여 자세한 내용을 확인할 수 있습니다. 이는 정보 과부하를 방지하고 사용자 경험을 개선합니다.
3.  **목표 부합:** "한눈에 정당 분포를 파악"하는 것은 지도 전체적인 관점에서 가장 강력한 정당의 분포를 파악하는 데 가장 효과적입니다. 세부적인 비율 비교는 툴팁으로 보완합니다.

### 구체적인 구현 방향

1.  **지도 데이터 처리:**
    *   각 광역/기초/재보궐 선거구 데이터에서 '가장 많은 의석을 차지한 정당'을 식별합니다. (예: `region.leaderPartyId`)
    *   각 정당에 고유하고 `#0a0e17` 다크 테마 배경에 잘 어울리는 색상 코드를 할당한 팔레트를 정의합니다.
        ```javascript
        const partyColors = {
            '민주당': '#36A2EB', // 밝은 파랑
            '국민의힘': '#FF6384', // 밝은 빨강
            '정의당': '#FFCE56', // 밝은 노랑
            '기타': '#C9CBCF', // 밝은 회색
            // ... 추가 정당 색상
        };
        ```
2.  **지도 색상 적용:**
    *   D3.js를 사용하여 지도 경계를 그릴 때, 각 `path` 요소의 `fill` 속성에 해당 지역의 `leaderPartyId`에 맞는 색상을 적용합니다.
    *   `path.attr('fill', d => partyColors[d.properties.leaderPartyId] || partyColors['기타'])`
    *   경계선(`stroke`)은 밝은 회색(`'#555'`)으로 얇게 설정하여 지역 구분을 명확히 합니다.
3.  **툴팁 구현 (D3 + Chart.js 또는 HTML List):**
    *   사용자가 지도 영역에 마우스를 올리면(또는 클릭 시) D3를 사용하여 `<div id="tooltip">` 요소를 생성하고, 마우스 위치 근처에 배치합니다.
    *   **툴팁 내용:**
        *   **옵션 1 (간소화된 도넛 차트 + 텍스트):** 툴팁 내부에 Chart.js로 최적화된 도넛 차트(위에서 제시된 개선안 적용)를 작게 표시하고, 그 아래에 지역 이름과 총 의석수를 텍스트로 보여줍니다. 이 도넛 차트는 애니메이션이 없고, `datalabels` 플러그인으로 백분율만 표시합니다.
        *   **옵션 2 (텍스트 리스트):** 툴팁 내부에 각 정당의 이름, 의석 수, 백분율을 목록(`<ul>` 또는 `<table>`) 형태로 깔끔하게 표시합니다. 각 정당 이름 옆에는 해당 정당의 색상(작은 사각형 또는 원)을 함께 표시하여 시각적 구분을 돕습니다.
            ```html
            <!-- 툴팁 내용 예시 -->
            <div style="background-color: #1a202c; color: #eee; padding: 10px; border-radius: 5px; border: 1px solid #555;">
                <h3>서울시 강남구</h3>
                <p>총 의석: 10석</p>
                <ul>
                    <li><span style="display:inline-block; width:10px; height:10px; background-color:#FF6384;"></span> 국민의힘: 6석 (60%)</li>
                    <li><span style="display:inline-block; width:10px; height:10px; background-color:#36A2EB;"></span> 민주당: 4석 (40%)</li>
                </ul>
            </div>
            ```
    *   툴팁은 마우스가 지도 영역에서 벗어나면 사라지도록(`mouseout`) 구현합니다.
    *   **다크 테마 툴팁 스타일:**
        ```css
        #tooltip {
            position: absolute;
            background-color: #1a202c; /* 어두운 배경 */
            color: #eee; /* 밝은 텍스트 */
            padding: 10px 15px;
            border-radius: 4px;
            border: 1px solid #444; /* 경계선 */
            pointer-events: none; /* 툴팁 위에서 마우스 이벤트가 발생하지 않도록 하여 하위 요소 클릭 방지 */
            opacity: 0;
            transition: opacity 0.2s ease-in-out;
            z-index: 1000;
        }
        #tooltip.active {
            opacity: 1;
        }
        ```
4.  **범례 (Legend):**
    *   지도 옆이나 하단에 각 정당 색상과 정당명을 매칭하는 범례를 반드시 표시해야 합니다. 이는 사용자가 지도를 해석하는 데 필수적입니다.
    *   다크 테마에 어울리게, 범례 배경은 어두운 색, 텍스트는 밝은 색으로 구성합니다.

이 권장안은 지도의 핵심 기능인 '지역적 분포 시각화'를 극대화하면서, 필요한 상세 정보를 효과적으로 제공하여 사용자가 지방선거 결과를 더욱 쉽게 이해할 수 있도록 도울 것입니다.
