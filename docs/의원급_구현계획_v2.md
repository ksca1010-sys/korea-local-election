## 1. 구조적 결정 6가지 (v1 대비 변경점)

### 결정 1: constituency를 districtName에 합쳐서 2단계 유지

```
v1 (위험): regionKey + districtName + constituencyName → 3단계
v2 (안전): regionKey + districtName → 2단계 유지

이유: App.selectRegion(regionKey, districtName, electionType) 시그니처를 
     바꾸면 기존 4종 선거 전체가 영향받음.

방법: districtName에 선거구명을 그대로 넣음.
  광역단체장: districtName = null (광역 단위)
  기초단체장: districtName = "종로구"
  광역의원:   districtName = "종로구제1선거구"  ← 선거구명이 들어감
  기초의원:   districtName = "종로구제1선거구"
  비례대표:   districtName = null (광역/기초 단위)

기존 코드 변경: 없음. selectRegion 시그니처 그대로.
```

### 결정 2: 선거종류 전환 시 districtName 초기화

```
문제: 기초단체장에서 "종로구" 선택 → 광역의원으로 전환
     → districtName이 "종로구"인데 광역의원에는 "종로구"라는 선거구가 없음

해결: setElectionType() 호출 시 districtName을 null로 초기화
     → 지도를 광역 레벨로 되돌림
     → 유저가 다시 선거구를 클릭하도록 유도

코드:
  App.setElectionType = function(newType) {
    const prevType = currentElectionType;
    currentElectionType = newType;
    
    // 선거 단위가 다르면 districtName 초기화
    const prevUnit = getElectionUnit(prevType);  // 'metro'|'district'|'constituency'
    const newUnit = getElectionUnit(newType);
    if (prevUnit !== newUnit) {
      currentDistrictName = null;
      MapModule.switchToProvinceMap(); // 전국 지도로 복귀
      showPlaceholder(); // "지역을 선택하세요" 표시
    }
    
    MapModule.setElectionType(newType);
  };
  
  function getElectionUnit(type) {
    switch(type) {
      case 'governor': case 'superintendent': return 'metro';
      case 'mayor': return 'district';
      case 'council': case 'localCouncil': return 'constituency';
      case 'councilProportional': return 'metro';
      case 'localCouncilProportional': return 'district';
      case 'byElection': return 'constituency';
      default: return 'metro';
    }
  }
```

### 결정 3: 의원급 렌더링을 별도 탭 파일로 분리

```
v1 (위험): app.js의 renderOverviewTab()에 8개 선거종류 분기
v2 (안전): js/tabs/council-tab.js로 분리

이유: app.js가 이미 5,128줄. 여기에 4종 분기를 더 넣으면 분기 지옥.
     history-tab.js가 이미 분리되어 있으니 같은 패턴 적용.

파일 구조:
  js/tabs/
    history-tab.js    (기존, 543줄)
    council-tab.js    (신규) — 광역의원 + 기초의원 지역구
    proportional-tab.js (신규) — 광역비례 + 기초비례

app.js에서의 호출:
  function renderCurrentTab(tabName) {
    const type = currentElectionType;
    
    // 의원급 지역구 → council-tab.js에 위임
    if (type === 'council' || type === 'localCouncil') {
      CouncilTab.render(tabName, currentRegionKey, currentDistrictName, type);
      return;
    }
    
    // 비례대표 → proportional-tab.js에 위임
    if (type === 'councilProportional' || type === 'localCouncilProportional') {
      ProportionalTab.render(tabName, currentRegionKey, currentDistrictName, type);
      return;
    }
    
    // 기존 로직 (governor, mayor, superintendent, byElection)
    switch(tabName) {
      case 'overview': renderOverviewTab(); break;
      case 'polls': renderPollTab(); break;
      ...
    }
  }

script 로드 순서 (index.html):
  <script src="js/tabs/history-tab.js"></script>
  <script src="js/tabs/council-tab.js"></script>       <!-- 신규 -->
  <script src="js/tabs/proportional-tab.js"></script>   <!-- 신규 -->
  <script src="js/app.js"></script>
```

### 결정 4: GeoJSON lazy-load (성능 보호)

```
v1 (위험): 기초의원 2,601개 선거구를 한 번에 로드
v2 (안전): 광역 클릭 시 해당 광역의 GeoJSON만 lazy-load

이유: 2,601개 선거구 GeoJSON을 전부 로드하면 수 MB → 모바일에서 버벅임.
     이미 data/basic_council/{region}/ 으로 광역별 폴더가 분리되어 있음.

방법:
  MapModule.loadConstituencyMap = async function(regionKey, electionType) {
    const folder = electionType === 'council' ? 'council' : 'basic_council';
    const url = `data/${folder}/${regionKey}/`;
    
    // 해당 광역의 GeoJSON 목록을 fetch
    // 이미 캐시된 것이면 재로드하지 않음
    if (this._constituencyCache[regionKey + electionType]) {
      return this._constituencyCache[regionKey + electionType];
    }
    
    const geojson = await fetch(`data/${folder}/${regionKey}.geojson`).then(r => r.json());
    this._constituencyCache[regionKey + electionType] = geojson;
    return geojson;
  };
```

### 결정 5: 대용량 후보자 JSON 광역별 분할

```
v1 (위험): local_council_candidates.json 하나에 20,000명
v2 (안전): 광역별 분할

파일 구조:
  data/candidates/council/           ← 광역의원
    seoul.json
    busan.json
    ...
  data/candidates/local_council/     ← 기초의원
    seoul.json
    busan.json
    ...

이유: 한 JSON에 20,000명 → 수 MB → 초기 로딩 지연.
     광역별로 분할하면 해당 광역 클릭 시에만 로드.

ElectionData 확장:
  ElectionData.loadCouncilCandidates = async function(regionKey) {
    if (this._councilCandidateCache[regionKey]) return;
    const data = await fetch(`data/candidates/council/${regionKey}.json`).then(r => r.json());
    this._councilCandidateCache[regionKey] = data;
  };
```

### 2.1 광역의원 (council) — 779개 선거구

#### 개요탭
```
선거구 정보:
  - 선거구명 (예: 서울 종로구 제1선거구)
  - 관할 읍면동 목록
  - 선출 인원: 1명 (소선거구)
  - 유권자 수

현직 의원:
  - 이름, 정당, 선수(N선)
  - 재출마 여부 (확인됨/미확인)

지난 선거 결과 (8회):
  - 바 차트 (후보별 득표율)
  - 투표율

선거 개요:
  - 있으면 표시, 없으면 상위 시군구 현안으로 폴백
```

#### 여론조사탭
```
99%의 선거구:
  "광역의원 선거구 단위의 여론조사는 거의 실시되지 않습니다.

1%의 격전지 (여론조사가 있는 경우):

```

  1. "{선거구명}" 검색
  2. "{시군구} 시의원/도의원" 검색
  3. "{광역시도} 광역의원" 검색

// data.js에 추가

// ─── 광역의원/기초의원 후보자 (광역별 lazy-load) ───
ElectionData._councilCandidateCache = {};

ElectionData.loadCouncilCandidates = async function(regionKey, electionType) {
  const folder = electionType === 'council' ? 'council' : 'local_council';
  const cacheKey = `${folder}_${regionKey}`;
  if (this._councilCandidateCache[cacheKey]) return;
  
  try {
    const data = await fetch(`data/candidates/${folder}/${regionKey}.json`).then(r => r.json());
    this._councilCandidateCache[cacheKey] = data;
  } catch(e) {
    this._councilCandidateCache[cacheKey] = { candidates: {} };
  }
};

ElectionData.getCouncilCandidates = function(regionKey, districtName, electionType) {
  const folder = electionType === 'council' ? 'council' : 'local_council';
  const cacheKey = `${folder}_${regionKey}`;
  const data = this._councilCandidateCache[cacheKey];
  if (!data?.candidates?.[districtName]) return [];
  return data.candidates[districtName].filter(c => c.status !== 'WITHDRAWN');
};

// ─── 비례대표 후보자 ───
ElectionData.loadProportionalCandidates = async function() {
  this.proportionalCandidates = await fetch('data/candidates/proportional.json').then(r => r.json());
};

ElectionData.getProportionalCandidates = function(regionKey, electionType) {
  const key = electionType === 'councilProportional' 
    ? 'council_proportional' 
    : 'local_council_proportional';
  return this.proportionalCandidates?.[key]?.[regionKey] || null;
};

// ─── 정당지지도 (비례대표용) ───
ElectionData.getPartySupport = function(regionKey) {
  // polls.json에서 type='정당지지도' AND region=regionKey 필터
  if (!this.pollsData?.regions?.[regionKey]) return [];
  return this.pollsData.regions[regionKey]
    .filter(p => p.classification?.electionTypes?.includes('정당지지도'));
};

// ─── 역대 결과 ───
ElectionData.loadCouncilHistory = async function() {
  this.councilHistory = await fetch('data/council_history.json').then(r => r.json());
};

ElectionData.getCouncilHistory = function(regionKey, districtName) {
  return this.councilHistory?.[regionKey]?.[districtName] || null;
