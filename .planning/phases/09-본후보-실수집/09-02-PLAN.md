# Plan 02: 수집 결과 검증 및 서비스 반영

## Objective
Plan 01에서 수집한 본후보 데이터의 불일치 항목을 수동 처리하고, 프론트엔드에서 기호순 정렬이 정상 동작하는지 확인한 후, 최종 배포하여 서비스에 반영한다.

## Context
Plan 01 실행 후 발견될 수 있는 3가지 불일치 유형(ballotNumber 필드명, 이름 불일치, wiwName 매핑 실패)에 대한 수동 처리 SOP(Standard Operating Procedure)다. 또한 5/15 18:00 KST에 자동 전환되는 기호순 정렬 모드가 정상 작동하는지 사전 확인한다. 프론트엔드 코드 변경은 불필요하다 -- `getCandidateSortMode()`가 타임스탬프 기반으로 자동 전환한다.

## Execution Date
**2026-05-14 오후 ~ 2026-05-15** -- Plan 01 완료 직후 실행, 기호순 정렬 확인은 5/15 18:00 이후

- [ ] Plan 01의 커밋이 main에 푸시되었다

### 1. ballotNumber 필드명 패치 (필요한 경우에만)

> Plan 01 Step 3에서 ballotNumber가 0명인 경우에만 실행

- [ ] **[확인]** raw 로그에서 실제 기호 필드명 찾기
  ```
  python3 -c "
  import json
  with open('data/candidates/nec_raw_sample.json') as f:
      data = json.load(f)
  known_fields = ['giho', 'gihoSn', 'candidateNo', 'huboNo']
  for typecode, sample in data.items():
      print(f'=== sgTypecode {typecode} ===')
      # 기존 4개 필드 확인
      for field in known_fields:
          if field in sample:
              print(f'  [FOUND] {field} = {sample[field]}')
      # 미지 필드 중 숫자값 찾기
      for key, val in sample.items():
          if key not in known_fields and isinstance(val, (int, str)) and str(val).isdigit() and int(str(val)) < 100:
              print(f'  [CANDIDATE] {key} = {val}  <-- 기호 필드 후보?')
  "
  ```

### 2. unmatched 후보 수동 처리 (필요한 경우에만)

> Plan 01 Step 5에서 unmatched > 0인 경우에만 실행

- [ ] **[확인]** unmatched 상세 내역
  ```
  python3 -c "
  import json
  with open('data/candidates/unmatched_candidates.json') as f:
      data = json.load(f)
  for i, item in enumerate(data.get('candidates', []), 1):
      c = item.get('candidate', {})
      print(f'{i}. {c.get(\"name\", \"?\")} ({c.get(\"jdName\", \"?\")})')
      print(f'   사유: {item.get(\"reason\", \"unknown\")}')
      print(f'   지역: {c.get(\"sdName\", \"?\")} / {c.get(\"wiwName\", \"?\")}')
      print()
  "
  ```

- [ ] **[판단]** 각 unmatched 후보에 대해 원인 분류
  - **A. 이름 불일치**: 예비후보와 본후보 이름 표기 차이 (한자, 띄어쓰기 등)
    - 조치: `status_updates.json`으로 교정
  - **B. wiwName 매핑 실패**: 기초단체장 선거구명이 기존 JSON 키와 불일치
    - 조치: 수동으로 올바른 키 확인 후 직접 JSON 편집
  - **C. 신규 후보**: 예비후보 없이 바로 본후보 등록한 신규 인물
    - 조치: 정상 -- 신규 추가 대상 (merge 함수가 자동 추가했어야 함, 안 됐다면 수동 추가)

- [ ] **[교정 A]** 이름 불일치 시 `status_updates.json` 편집 (광역단체장만 적용됨)
