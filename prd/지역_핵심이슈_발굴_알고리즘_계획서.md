# 지역 핵심이슈 발굴 알고리즘 설계 계획서

> 작성일: 2026.03.09 | 대상: 6·3 선거정보지도 — 지역별 이슈 카드 (최대 6개, 최소 0개)

---

## 1. 설계 철학

**"없으면 안 보여준다."**

이슈는 0~6개 사이에서 유동적으로 결정된다. 진짜 이슈만 올라와야 서비스의 신뢰도가 유지되므로, 점수가 임계값(threshold)을 넘기지 못하면 빈 상태로 두는 것이 원칙이다. 허수 이슈를 채워 넣어 6개를 맞추는 것은 금지.

```
이슈 후보 풀 → 점수화 → 임계값 필터 → 중복 제거 → 상위 N개 (N ≤ 6)
                                          ↓
                                    N = 0이면 "이 지역의 주요 이슈를 분석 중입니다" 표시
```

---

## 2. 이슈 발굴 소스 — 5개 채널

각 채널은 독립적으로 이슈 후보를 생성하고, 최종 단계에서 통합 점수로 합산된다.

### 채널 ① 뉴스 언급빈도 분석 (News Frequency)

**원리**: 특정 지역 + 특정 기간 내 뉴스 기사에서 반복 등장하는 키워드 클러스터를 이슈로 추출

```
수집: "{지역명}" 포함 뉴스 기사 최근 30일분
  → 형태소 분석(명사 추출)
  → 불용어 제거 (정치인 이름, 정당명, 일반 선거 용어)
  → TF-IDF 또는 단순 빈도 카운트
  → 상위 키워드 클러스터링 (유사 키워드 묶기)
  → 클러스터 = 이슈 후보 1건
```

**구체 예시**:
```
지역: 인천 연수구
30일간 뉴스 키워드 빈도:
  "GTX-B" (47회), "수인선 환승" (31회) → 클러스터: "GTX-B 교통 인프라"
  "아파트 재건축" (38회), "노후계획도시" (25회) → 클러스터: "노후 아파트 정비"
  "인구유출" (12회) → 빈도 낮아 임계값 미달 → 탈락
```

**데이터 소스**: 네이버뉴스 검색 API, 다음뉴스 API, 지역신문 RSS

**강점**: 실시간성이 높음, 기초지역도 지역신문을 통해 커버 가능
**약점**: 언론이 다루지 않는 생활밀착 이슈는 놓칠 수 있음

---

### 채널 ② 검색 트렌드 분석 (Search Trend)

**원리**: 해당 지역 주민들이 실제로 무엇을 검색하는지로 관심 이슈를 역추적

```
네이버 데이터랩 Search Trend API:
  → 후보 키워드 그룹 설정: "{지역명} + {이슈 키워드}"
  → 기간별 검색량 추이 조회 (연령별, 성별 세분화 가능)
  → 최근 급상승(spike) 키워드 = 이슈 후보
```

**후보 키워드 씨앗(seed) 목록**:
```javascript
const ISSUE_SEEDS = [
  // 교통·인프라
  "교통", "도로", "지하철", "버스", "주차", "GTX", "KTX", "철도",
  // 부동산·주거
  "재건축", "재개발", "아파트", "분양", "집값", "임대",
  // 교육
  "학교", "학군", "보육", "어린이집", "교육", "급식",
  // 환경
  "미세먼지", "쓰레기", "소음", "악취", "환경", "폐기물", "태양광",
  // 경제·일자리
  "일자리", "고용", "산업단지", "창업", "폐업", "상권",
  // 복지·의료
  "병원", "응급실", "요양", "복지", "출산", "인구감소",
  // 안전
  "범죄", "CCTV", "재난", "침수", "산불",
  // 문화·관광
  "관광", "축제", "문화시설"
];
```

**활용 방식**:
```
"{영암군} {태양광}" 검색 트렌드 → 최근 30일 급상승 → 이슈 후보
"{인천} {GTX-B}" 검색 트렌드 → 지속적 고관심 → 이슈 후보
"{평택} {고덕신도시}" 검색 트렌드 → 상승 추이 → 이슈 후보
```

**강점**: 주민의 실제 관심사를 반영, 언론 사각지대 이슈도 포착 가능
**약점**: 검색량이 적은 농어촌 기초지역에서는 데이터 부족

### 채널 ③ 여론조사 이슈 항목 추출 (Poll Issues)

**원리**: 여심위 등록 여론조사 중 "가장 중요한 지역 현안" 문항의 응답을 직접 활용

```
여심위(nesdc.go.kr) 등록 여론조사에서:
  → "이번 선거에서 가장 중요하게 생각하는 지역 현안은?" 문항 추출
  → 응답 항목과 비율을 그대로 이슈 후보로 등록
```

**구체 예시**:
```
[인천시장 여론조사 - 한국갤럽 2026.03.05]
Q. 인천시장을 뽑을 때 가장 중요하게 고려하는 지역 현안은?
  1. 교통 인프라 개선 (32%)
  2. 노후 주거지역 정비 (24%)
  3. 일자리 창출 (18%)
  4. 교육 환경 개선 (14%)
  5. 환경·안전 (8%)
  6. 기타 (4%)

→ 32% "교통 인프라 개선" → 이슈 후보 (응답률 자체가 점수)
→ 24% "노후 주거지역 정비" → 이슈 후보
→ 8% "환경·안전" → 응답률 10% 미만 → 임계값 미달 → 탈락
```

**강점**: 가장 직접적인 유권자 의견, 신뢰도 최고 (여심위 등록 데이터)
**약점**: 모든 지역에 여론조사가 있지 않음 (광역 위주), 기초지역은 희소

---

### 채널 ④ 지자체 민원·청원 데이터 (Civic Data)

**원리**: 지자체에 접수된 민원·국민청원 데이터에서 빈출 키워드를 추출하여 생활밀착 이슈 발굴

```
소스:
  ├─ 국민신문고(epeople.go.kr) 민원 통계
  ├─ 각 지자체 민원시스템 공개 데이터
  ├─ 국민동의청원(petitions.assembly.go.kr) 지역별 청원
  └─ 지방의회 5분자유발언, 질의서 텍스트

처리:
  → 최근 6개월 민원 키워드 빈도 분석
  → 급증 민원 카테고리 = 이슈 후보
```

**구체 예시**:
```
영암군 민원 데이터 (최근 6개월):
  "태양광 설치 민원" (87건) → 이슈 후보: "태양광 발전 갈등"
  "도로 포장 요청" (54건) → 이슈 후보: "도로 인프라"
  "의료기관 부족" (41건) → 이슈 후보: "의료 접근성"
  "가로등 설치" (12건) → 빈도 낮음 → 탈락
```

**강점**: 언론이 다루지 않는 생활밀착·기초 지역 이슈 포착에 매우 강력
**약점**: 공공데이터 개방 수준이 지자체마다 다름, 실시간성 낮음 (월/분기 단위 갱신)

---

### 채널 ⑤ 지역 커뮤니티 버즈 분석 (Community Buzz)

**원리**: 지역 온라인 커뮤니티에서 화제되는 토픽을 추출

```
소스:
  ├─ 지역 맘카페 (네이버 카페: "인천맘", "수원맘" 등)
  ├─ 지역 커뮤니티 (네이버 카페: "인천사랑", "전남사랑" 등)  
  ├─ 에브리타임 (대학생 지역 커뮤니티)
  ├─ 블라인드 지역 게시판
  └─ X(Twitter) 지역 해시태그

처리:
  → 게시글 제목+본문 수집 (공개 게시판만, 비공개 제외)
  → 최근 2주간 게시글 키워드 빈도 + 댓글수 가중
  → 급상승 토픽 = 이슈 후보
```

**강점**: 실시간 민심 반영, 언론·행정 사각지대 이슈 포착
**약점**: 노이즈가 많음 (광고, 중복, 감정적 글), 크롤링 허용 범위 제한

## 3. 통합 점수 산출 알고리즘

### 3.1 각 채널별 이슈 후보 → 정규화 점수 (0~1)

```javascript
// 각 채널의 raw 값을 0~1로 정규화
function normalize(value, min, max) {
  if (max === min) return 0;
  return Math.min(1, Math.max(0, (value - min) / (max - min)));
}

// 채널별 정규화 방법
const channelScores = {
  news:      normalize(언급횟수, 0, 해당지역_최대언급횟수),
  search:    normalize(검색트렌드_상승률, 0, 300),  // 300% 이상이면 만점
  poll:      normalize(응답률, 0, 100),              // 응답률 그대로 (%)
  civic:     normalize(민원건수, 0, 해당지역_최대민원건수),
  community: normalize(게시글수×댓글가중, 0, 해당지역_최대버즈)
};
```

### 3.2 통합 점수 — 가중 합산

```javascript
function calcIssueScore(channelScores, availability) {
  // 가중치 (데이터가 있는 채널만 참여, 동적 재배분)
  const WEIGHTS = {
    news:      0.30,  // 뉴스 언급빈도
    search:    0.20,  // 검색 트렌드
    poll:      0.25,  // 여론조사 (있으면 가장 신뢰)
    civic:     0.15,  // 민원·청원
    community: 0.10   // 커뮤니티 버즈
  };
  
  // 데이터가 없는 채널의 가중치를 나머지에 재배분
  let totalWeight = 0;
  let weightedSum = 0;
  
  for (const [channel, weight] of Object.entries(WEIGHTS)) {
    if (availability[channel]) {  // 이 채널에 데이터가 있는가?
      totalWeight += weight;
      weightedSum += weight * channelScores[channel];
    }
  }
  
  if (totalWeight === 0) return 0;  // 어떤 채널에도 데이터 없음
  return weightedSum / totalWeight;  // 정규화된 최종 점수 (0~1)
}
```

### 3.3 임계값 필터 — "허수 제거"의 핵심

```javascript
const THRESHOLD = 0.25;  // 최종 점수 0.25 미만이면 이슈로 인정하지 않음

function filterIssues(issueCandidates) {
  return issueCandidates
    .filter(issue => issue.finalScore >= THRESHOLD)
    .filter(issue => issue.dataChannelCount >= 2)  // 최소 2개 채널에서 감지
    .sort((a, b) => b.finalScore - a.finalScore)
    .slice(0, 6);  // 최대 6개
}
```

**핵심 규칙**:
- 최종 점수 **0.25 미만** → 탈락 (허수 방지)
- **2개 이상 채널**에서 동시에 감지되어야 이슈로 인정 (단일 소스 노이즈 제거)
- 통과한 이슈가 0개면 → 0개 표시 (강제 채우기 금지)

## 5. 광역 vs 기초 — 채널 우선순위 차이

```
┌─────────────────────────────────────────────────────────────────┐
│  광역 단위 (시·도지사, 교육감)                                    │
│                                                                 │
│  [1순위] 여론조사 (0.30)  — 광역 여론조사 풍부                    │
│  [2순위] 뉴스 언급 (0.25) — 전국지+지역일간지 커버리지 높음       │

│  [4순위] 민원데이터 (0.15) — 광역 단위 민원 집계                  │
│  [5순위] 커뮤니티 (0.10)  — 보조 검증용                           │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  기초 단위 (구시군장, 기초의원)                                    │
│                                                                 │
│  [1순위] 뉴스 언급 (0.30)  — ★지역신문이 핵심 소스★              │
│  [2순위] 민원데이터 (0.25) — 기초 단위에서 가장 직접적            │
│  [3순위] 커뮤니티 (0.20)  — 맘카페·지역카페가 생활이슈 포착       │
│  [4순위] 검색트렌드 (0.15) — 데이터 부족할 수 있음                │
│  [5순위] 여론조사 (0.10)  — 기초 여론조사는 희소 (있으면 보너스)  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 6. 이슈 발굴 방법론 — 실전 제안 7가지

이슈 발굴은 알고리즘만으로 완결되지 않는다. 아래는 **바이브코딩 수준에서 실제 구현할 수 있는 방법론** 7가지다.

### 방법 1. 뉴스 키워드 클러스터링 (자동)

```
작동: 지역명으로 뉴스 검색 → 명사 추출 → TF-IDF → 상위 키워드 군집화
도구: 네이버뉴스 검색 API + 형태소 분석기 (Okt/Mecab)
난이도: ★★☆ (API 호출 + 텍스트 처리)
커버리지: 모든 지역
```

### 방법 2. 네이버 데이터랩 검색 급상승 추적 (자동)

```
작동: 이슈 씨앗 키워드 × 지역명 조합을 주기적으로 Search Trend API에 조회
     → 전주 대비 200% 이상 급상승한 키워드 = 이슈 후보
도구: 네이버 Search Trend API (일 1,000건 무료)
난이도: ★☆☆ (API 호출만으로 가능)
커버리지: 검색량이 있는 지역 (도시 > 농촌)
```

### 방법 3. 여론조사 현안 문항 자동 수집 (반자동)

```
작동: 여심위 등록 여론조사 중 "지역 현안" 관련 문항 답변을 주기적 수집
     → 응답률 10% 이상 항목을 이슈로 등록
도구: 여심위 홈페이지(nesdc.go.kr) 크롤링 또는 수동 입력
난이도: ★★☆ (크롤링) / ★☆☆ (수동)
커버리지: 여론조사가 실시된 지역만 (광역 위주)
```

### 방법 4. 지방의회 회의록 텍스트 마이닝 (자동)

```
작동: 지방의회 회의록 공개 데이터에서 최근 6개월 키워드 빈도 분석
     → 의원들이 반복적으로 질의하는 주제 = 지역 핵심 현안
도구: 지방의회 회의록 시스템 (council.go.kr 등)
난이도: ★★★ (데이터 정제 필요)
커버리지: 전국 모든 기초·광역 의회
특장점: 가장 공식적이고 체계적인 지역 현안 데이터
```

### 방법 5. 민원 통계 분석 (반자동)

```
작동: 국민신문고 민원 통계 또는 지자체 공개 민원 데이터에서
     → 최근 급증 민원 카테고리를 이슈로 추출
도구: 국민신문고 API, 정보공개포털(open.go.kr)
난이도: ★★☆
커버리지: 전국 (단, 데이터 갱신 주기가 느림)
```

### 방법 6. LLM 기반 이슈 요약 (자동)

```
작동: 채널 ①~⑤에서 수집된 raw 데이터를 LLM에 넘겨
     → "이 지역의 핵심 선거 이슈를 3~6개로 요약해줘" 프롬프트
     → LLM이 키워드 군집화 + 자연어 이슈명 생성
도구: Claude API (Sonnet — 비용 효율적)
난이도: ★☆☆ (프롬프트만 잘 짜면 됨)
커버리지: raw 데이터가 있는 모든 지역
특장점: 바이브코딩에 가장 적합, 별도 NLP 파이프라인 불필요
```

### 방법 7. 수동 큐레이션 보정 (수동)

```
작동: 알고리즘이 놓치는 이슈를 편집자가 수동 추가/삭제/보정
     → 특히 지역 전문가(지역 활동가, 기자)의 피드백 반영
     → 관리자 페이지에서 이슈 카드 수동 편집 가능
난이도: ★☆☆
커버리지: 편집자가 아는 지역
특장점: 알고리즘의 맹점을 보완하는 최후의 안전망
```

## 7. 추천 구현 전략 — 바이브코딩용 우선순위

리소스가 제한된 상황에서 **가성비 최고 조합**:

```
[MVP — 즉시 구현 가능]
  방법 6 (LLM 이슈 요약) + 방법 1 (뉴스 키워드)
  → 지역명으로 뉴스 검색 → 결과를 Claude API에 넘겨 이슈 요약
  → 구현 시간: 반나절

[Phase 2 — 1주 내 추가]
  방법 2 (검색 트렌드) + 방법 3 (여론조사)
  → 검색 트렌드로 교차 검증 + 여론조사 데이터 있으면 최우선 반영

[Phase 3 — 선거 전 고도화]
  방법 4 (의회 회의록) + 방법 5 (민원)
  → 공식 데이터로 이슈 깊이 보강

[상시 운영]
  방법 7 (수동 큐레이션) — 알고리즘 보정용 관리자 기능
```
