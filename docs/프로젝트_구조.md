## 1. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 프론트엔드 | 바닐라 JS + D3.js v7 + Chart.js v4 | 지도·차트·UI |
| 스타일 | CSS3 (반응형) + Font Awesome + Noto Sans KR | 디자인 |
| 데이터 | JSON (정적 호스팅) | Git 추적 가능 |
| 자동화 | Python 3.11 + GitHub Actions | 일일 배치 |
| LLM | Claude Haiku (개요/팩트체크/공약수집) | 자동 분석 |
| API | 네이버 뉴스, NESDC, NEC(선관위) 예비후보·개표결과 | 외부 데이터 |
| 뉴스 프록시 | Cloudflare Workers | 네이버 뉴스 API CORS 우회 |
| 배포 | Cloudflare Pages | 정적 호스팅 |

## 3. 데이터 동기화 시스템

### 필터 카운트 + 전국개황 연동

```
데이터 로드 완료 시점
  │
  ├── syncCountsFromData()          ← 실제 데이터에서 동적 계산
  │   ├── regions 객체 키 수        → 광역단체장/교육감 수 + 전국개황 시도 수
  │   ├── subRegions 시군구 합계    → 기초단체장 수 + 전국개황 시군구 수
  │   └── byelection.json 키 수     → 재보궐 수
  │
  ├── updateFilterCounts()          ← 필터 버튼 숫자 갱신
  │
  └── renderStats()                 ← 전국개황 카드 갱신
```

광역 합병 등 행정구역 변경 시 데이터(regions.json)만 수정하면 UI가 자동 반영됨.

### 재보궐 파이프라인 상세 (fetch_byelection.py)

```
① 선관위 예비후보 API (공식 데이터, 최우선)
   └── 신규 후보 추가 / 당적 교정 / RUMORED→DECLARED 승격
②  뉴스 기반 Claude 분석 (출마설, 사퇴 등)
③ 뉴스 교차검증 (당적 2차 확인)
④ 당적 일관성 검증 (party/partyKey/partyName 동기화)
⑤ 저장 + 자동 커밋
```

## 7. 뉴스 시스템 아키텍처

### 프록시
```
프론트엔드 → Cloudflare Worker → 네이버 뉴스 API
              https://election-news-proxy.ksca1010.workers.dev
```

### 5차원 스코어링
```
점수 = (시의성 × 25%) + (관련성 × 20%) + (신뢰도 × 10%) + (지역성 × 35%) + (인기도 × 10%)
```

### 선거종류별 뉴스 필터
| 선거종류 | 카테고리 | 특수 처리 |
|---------|---------|----------|
