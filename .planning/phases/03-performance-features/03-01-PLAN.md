# Plan 03-01: 대용량 JSON 지연 로딩 + esbuild 번들 + ESLint 0건

- `package.json`에 `"build"` 스크립트가 존재한다
- `deploy.sh`에 esbuild 빌드 단계가 포함된다
- `eslint.config.js`가 존재하고 `npm run lint`가 0 오류로 통과한다

**QUAL-05: esbuild 번들**

1. `npm install --save-dev esbuild` 실행

2. `package.json`에 build 스크립트 추가:
   ```json
   "build": "node build.js"
   ```

3. 프로젝트 루트에 `build.js` 생성:
   - esbuild를 사용하여 `js/` 하위 모든 JS를 단일 번들로 결합
   - 방식: `index.html`의 `<script defer>` 순서대로 entryPoints 나열하는 대신, 바닐라 JS IIFE 패턴이므로 **concatenate + minify** 접근 사용
   - `esbuild.build({ entryPoints: [...all js files in order], bundle: false, minify: true, outdir: '.deploy_dist/js/', ... })`
   - 또는 각 파일을 개별 minify 후 같은 구조 유지 (CDN script 태그 호환)
   - **중요:** 이 프로젝트는 ES modules가 아닌 전역 IIFE 패턴 — `bundle: false` + `minify: true`로 각 파일 개별 미니파이
   ```js
   const esbuild = require('esbuild');
   const glob = require('node:fs').readdirSync;
   // js/ 하위 모든 .js 파일을 개별 minify
   esbuild.buildSync({
     entryPoints: [/* index.html 순서대로 모든 js/*.js, js/tabs/*.js, js/views/*.js */],
     outdir: '.deploy_dist/js',
     minify: true,
     bundle: false,
     target: ['es2020'],
   });
   ```

4. `deploy.sh` 수정 — rsync 후 esbuild minify 단계 삽입:
   - rsync로 `.deploy_dist/` 생성 후, `node build.js` 실행하여 `.deploy_dist/js/` 내 파일을 minified 버전으로 덮어쓰기
   - 위치: `rsync` 명령 이후, `wrangler pages deploy` 이전
   ```bash
   echo "Minifying JS..."
   node build.js || { echo "esbuild minify 실패 — 배포 중단"; exit 1; }
   ```

5. `css/style.css`도 esbuild로 minify 추가 (build.js에 CSS entryPoint 추가)
